梁面・アウトライン抽出群 gh_beam_face_extractor

梁のソリッドから上下フランジ面・ウェブ面・中心線・断面を一発で取り出す下ごしらえ部品。

稼働中 GHパイソン群（4ファイル） Rhino7/8 両対応

MOTION PREVIEW

動作デモ

PREVIEW⟲ LOOP 10s

梁ソリッドを入力 → 上フランジ上面・ウェブ面・下フランジ面を順に抽出し中心線を通す → 端部の断面カーブ・ウェブの中心面・頂点まで分解する、という流れを10秒でループ再生しています。

1なにをするツール？

H形鋼などの梁のソリッド（Brep／メッシュ）を、部位ごとの面や線に分解して取り出すGHコンポーネント群です。上フランジの上面・下フランジの下面・ウェブ（腹板）の両面・部材の中心線・ウェブ中間の中心面・断面カーブ・エッジ・頂点まで、欲しい要素を端子から個別に受け取れます。

「梁のこの面を基準に何かを作りたい」「溶接線や接合部の位置を面から拾いたい」「かさ上げや配置の基準面が欲しい」といったモデリングの下ごしらえに使う、地味だが便利な部品箱です。ブロックインスタンス（アセンブリ）に入った梁にも対応します。

※ フランジ＝H形鋼の上下の板。／ ウェブ＝上下フランジをつなぐ縦の板（腹板）。／ 中心線＝部材の長手方向を示す軸。

3使い方

  1. 梁を渡す抽出したい梁のGUID（またはジオメトリ）を ID へ。複数まとめて渡してもよい。ブロック内の梁もそのまま。
  2. 欲しい端子だけ使う必要な面・線だけを下流につなぐ。例: 上面基準なら TopFace、軸が欲しければ CenterLine。
  3. 傾いた梁は oriented 版斜め・回転した梁で面の割り当てが乱れるときは gh_beam_face_extractor_oriented に切り替えると安定する。
  4. 上面の外形が欲しいときは beam_outline複数梁の天端をひとつの外形線・面として取りたい場合は beam_outline に梁群を渡す（合体してから外形化）。

4つながり（ワークフロー）

単独で使えます。 どのワークフローにも属さない汎用の「下ごしらえ部品」です。取り出した面・中心線・断面を、かさ上げ鋼材・接合部・寸法基準など自分の目的の入力として使ってください。

5自分のモデルで試す

専用サンプルはありません。H形鋼の梁が1本あればすぐ試せます（柱梁フレームのモデルなどでOK）。

  1. H形の梁を1本用意Brepソリッドの梁を1つ選ぶ。ブロックに入っていても可。
  2. IDを渡して solveそのGUIDを ID に。TopFace や CenterLine に結果が出る。
  3. 面をプレビュー確認各端子をパネル/プレビューで見て、狙った面が正しく取れているか確認する。

6注意点・既知の罠

  • ⚠ 面の割り当てはH形（フランジ＋ウェブ）前提

    上フランジ・下フランジ・ウェブという分類はH形断面を前提にしています。BOX・パイプ・不整形な断面では面ラベルが期待とずれることがあります。その場合は AllFaces から自分で選ぶのが安全です。

  • ⚠ 軸の自動検出は最長エッジ／主成分に依存

    部材の長手軸は「最長の直線エッジ」または頂点群の主成分（PCA）から推定します。極端に短い材・曲がり材・端部形状が複雑な材では軸を取り違えることがあります。うまくいかないときは oriented 版を試してください。

  • ⚠ Py3専用構文は避ける（Rhino7互換）

    beam_outline はRhino7（IronPython 2.7）／Rhino8（Python3）両対応の互換シムを持ちます。改造時に f-string や := を入れるとRhino7側で止まります。周囲のスタイルに合わせてください。
