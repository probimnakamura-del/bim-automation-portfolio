図面化パイプライン (Make2D)

断面符号＆タイトル抽出 GH_SectionSymbolTitle

図面の「A-A」などの断面符号と切断図タイトルを、自動で見つけてペアにする。

稼働中 大文字専用（誤検知防止・厳格版） GH Python ノード

MOTION PREVIEW

動作デモ

PREVIEW⟲ LOOP 10s

図面上の丸で囲まれた断面符号（A-A）と、対応する切断図タイトル（断面図 A-A）をRunで見つけ出し、符号とタイトルをペアにして抽出リストへ積み上げる流れを10秒でループ再生しています。

1なにをするツール？

製作図に散らばっている大文字の断面符号（A-A、B-Bなど）と、それに対応する切断図タイトルを一括で抽出するツールです。断面符号は「丸で囲まれた文字」として図面に置かれ、切断位置を示す水平線や、どちら向きに見るかを示す矢視とセットになっています。このノードは、それらを見つけ出して符号（例：A-A）と切断図タイトル（例：断面図 A-A）をペアで束ねます。

ポイントは、英単語の中の連続文字（"TT" や "SS" など）をタイトルと誤認しないよう、厳格な誤検知防止ルールで拾っていること。それでも図面の書き方によっては取りこぼしが出るので、AddIds 端子に断面符号ブロックのGUIDを渡せば、ルールを無視して強制的に断面符号として追加できます。ここで拾った断面符号は、下流の 統合JSON生成 に渡り、Make2Dの切断計画の材料になります。なお大文字専用で、小文字の断面（a, b…）は姉妹ツール section_lower が別ルールで担当します。

※ 断面符号＝「この位置でこう切って、この方向から見る」を表す図面記号（A-A など）。／ 切断図タイトル＝その断面図に付く見出し文字（「断面図 A-A」など）。両者をペアにすることで、どの符号がどの断面図に対応するかが機械的に分かります。

3使い方

  1. 図面を開いてノードを置く断面符号（丸囲みのA-Aなど）とタイトルが入った2D図面をRhinoで開き、GHにこのノードを配置する。
  2. Runで抽出Run に Button を付けて押す。断面符号と切断図タイトルが抽出され、符号側・タイトル側の各端子に出力される。
  3. 符号とタイトルの対応を確認Names と Title_Texts を見て、A-A・B-B…がそれぞれのタイトルと揃っているか確認する。
  4. 取りこぼしを手動追加拾えなかった符号があれば、ピッカーでブロックのGUIDを取って AddIds へ流す（ルール無視で強制追加）。
  5. 追加分を目で確かめるSelectAdded をオンにすると、追加した符号の元オブジェクトがRhinoで選択され、何を足したか一目で分かる。
  6. 統合JSONへ渡す符号・タイトルを GH_SectionDataJSON に接続し、通り芯・レベル線と束ねて統合JSONにする。

4つながり（ワークフロー）

前工程 なし（2D図面から直接抽出する起点ノード）
↓ GH_SectionSymbolTitle（このノード） ↓
後工程 統合JSON生成 GH_SectionDataJSON

兄弟ツール：小文字断面（a, b…）は section_lower が担当。通り芯は grid_extractor_v2。これらが 統合JSON に集約されます。全体像は Make2Dパイプライン詳細図 へ。

5サンプルモデルでデモ

📐 サンプルモデル\02_2D図面.3dm

通り芯記号（X1, X2, Y1…）・断面符号（A-A）・タイトル文字入りの2D図面。断面符号抽出・通り芯抽出・統合JSONのデモ用。

  1. 図面を開くサンプルモデル\02_2D図面.3dm を開く。
  2. ノードを置いてRunGHにこのノードを置き、Run のボタンを押す。断面符号 A-A と切断図タイトルが抽出される。
  3. 符号とタイトルを確認Names に「A-A」、Title_Texts に「断面図 A-A」が出て、ペアになっていることを確認。
  4. 統合JSONへつなぐ同じ図面から 通り芯 も抽出し、両方を 統合JSON生成 へ渡すとパイプラインの入口が完成する。

6注意点・既知の罠

  • ⚠ 大文字専用。小文字はsection_lowerへ

    このノードが扱うのは大文字の断面符号（A-A、B-B…）だけです。小文字断面（a, b, g…）は取得ルールが異なるため、専用の section_lower パイプラインが担当します。混在図面では両方を使い分けてください。

  • ⚠ 取得ルールは厳格。取りこぼしはAddIdsで救済

    英単語内の連続文字（"TT" "SS" 等）をタイトルと誤認しないよう、あえて厳しめに拾います。その分、変則的な符号は自動で拾えないことがあります。自動ルールは崩さず、取りこぼしだけ AddIds にブロックGUIDを渡して強制追加してください。

  • ⚠ Runは必ずButtonで・結果はドキュメント単位で保持

    Run は押した時だけ抽出する Button 運用が基本。結果は「このコンポーネント × このドキュメント」ごとに保持され、図面を開き直すと「未計算」に戻るので、その時はもう一度押してください。
