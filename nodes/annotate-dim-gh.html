図面化パイプライン (Make2D)

寸法注記GH版 GH_AnnotateDim

範囲を2点で囲むだけ。AIが寸法の入れ方を提案し、あなたのクセを覚えていく注記ツール。

稼働中 v1.0 GH Python3 ノード

MOTION PREVIEW

動作デモ

PREVIEW⟲ LOOP 10s

範囲を2点で囲む → AIが点線で寸法を提案 → Enterで実線に確定して学習、という流れを10秒でループ再生しています。

1なにをするツール？

図面に寸法線と文字（注記）を素早く入れるためのGrasshopperノードです。Rhinoコマンド版の annotate_dim_text の作図機能をGHに移植したもので、連続寸法・向きの自動判定・注記どうしの被り回避を備えています。

いちばんの特徴はAI寸法提案（AIDim）。寸法を入れたい範囲を2点で囲むと、その中の注記テキストや形状の特徴をAIへ渡し、「ここにこう寸法を並べる」という寸法チェーンを提案してくれます。提案はまず点線のプレビューで出るので、良ければEnterで実線に確定。確定した内容は annodim_examples.jsonl に学習データとして蓄積され、次からの提案（few-shot）に活かされるので、使うほどあなたの寸法の入れ方のクセに近づいていきます。

※ AIは既定でパソコン内のローカルAI（Ollama）を使います。環境変数を切り替えるとClaude APIも使えます。／ Make2Dの断面（SecNames/SecXf/SecBox）を受け取れるので、GH_Make2DExecutor が出した図面にそのまま寸法を入れる、という連携が想定されています。

3使い方

  1. 図面に対して置くMake2Dで断面を起こした図面のGHに本ノードを置く。GH_Make2DExecutor の SecNames/SecXf/SecBox/Target3dm をつなぐと、断面ごとに正しい位置へ寸法が入る。
  2. 手で入れるか、AIに任せるか選ぶ1本ずつなら PickDim / PickTxt で連続作図。まとめて欲しいときは AIDim を押す。
  3. 範囲を2点で囲むAIDimでは、寸法を入れたい範囲を2点でクリック。中の注記や形状の特徴をAIが読み取る。
  4. 点線プレビューを確認提案された寸法チェーンが PrevCrvs に点線で表示される。位置・本数をチェック。
  5. Enterで確定良ければ確定して実線の寸法に。内容は annodim_examples.jsonl に学習として蓄積され、次回以降の提案が賢くなる。
  6. 整えて書き戻す被りやズレは EditDim で調整。仕上がったら Bake または WriteBack で図面へ反映。

5実データでためす

📐 専用サンプルはなし — Make2D出力の図面上で使います

本ノードは「Make2Dで起こした断面図に寸法を入れる」ためのものなので、まず断面を出してから試すのが分かりやすいです。

  1. 断面を出すサンプルモデル\01_柱梁フレーム.3dm を GH_Make2DExecutor で1〜2面Make2Dし、図面座標に2D断面線を出す。
  2. 本ノードをつなぐExecutorの SecNames/SecXf/SecBox を本ノードへ配線。
  3. AIDimで提案させるAIDim を押し、断面を2点で囲む。点線の寸法提案が出る（ローカルOllamaが起動している必要あり）。
  4. 確定して学習良い提案をEnterで確定。数回繰り返すと、あなたの寸法の入れ方に近い提案が返るようになる。

6注意点・既知の罠

  • ⚠ AI提案にはローカルAI（Ollama）が要る

    AIDimは既定でパソコン内のローカルAI（Ollama）を呼びます。Ollamaが起動していないと提案が返りません。ネット越しのClaude APIを使いたい場合は環境変数で切り替えます（APIキー設定が必要）。AIを使わない手動の寸法・文字入れは、AIなしでもそのまま使えます。

  • ⚠ 提案は必ず点線プレビューで確認してから確定

    AIの提案が常に完璧とは限りません。PrevCrvs の点線を見て、本数・位置がおかしくないか確認してからEnterで確定してください。確定した内容がそのまま学習データになるため、変な提案をそのまま通すと学習も濁ります。

  • ⚠ RH8のPython3ノード（IronPythonでは動きません）

    本ノードはRhino8の新しいPython3コンポーネント前提です。Rhino7の旧Python（IronPython 2.7）では動きません。

  • ⚠ 回帰テストは緑だが実機での作図は要確認

    回帰テスト（test_annodim.py・24項目）は通っていますが、実際の図面での作図結果は目で確かめてから採用してください。特に注記の被り回避は形状によって調整が要ることがあります。
