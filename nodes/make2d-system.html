動作デモ

PREVIEW⟲ LOOP 10s

BuildボタンひとつでMake2Dパイプラインの9ノードと配線・グループ枠が一括生成され、さらにキャンバスを触らずに使える操作パネルが立ち上がる様子を10秒でループ再生しています。

1なにをするツール？

Make2D図面化パイプラインは9個ものノードを正しい順番でつなぐ必要があり、毎回手で並べて配線するのは大変です。このツールは、その面倒な組み立てをボタン1つで肩代わりします。空のGrasshopperキャンバスにこのノードを置いて Build を押すだけで、通り芯交点抽出・プランナー・IFC属性検索・接触検出・追加/除外ピッカー・最終選択・Make2D実行・部材記号・寸法注記の全ノードと配線、グループ枠まで一気に生成されます。

各ノードは「正本の.pyを1行で読み込むスタブ（ローダー）」として置かれるので、あとからコードを直したいときは C:\Python1\図面整理\ の正本を編集するだけでOK。全ノードに貼り直す必要はありません。さらに、Grasshopperのキャンバスを開かなくても操作できる常時表示のコントロールパネルもポップアップし、「通り芯を拾う→断面を決める→Make2D→書き戻す」の手順をパネル上のボタンから順に実行できます。

※ スタブ（ローダー）＝本体のコードを別ファイルから読み込むだけの短いつなぎ。中身は正本1か所に集約されるので、更新が全ノードへ一発で反映されます。／ このページで扱う旧配布形態「Make2D共有版.zip」については 下の注記 を参照してください。

2入出力（GHノードの端子）

入力ポート（2）
端子説明
Buildこれがメインのボタン。押すとMake2Dパイプライン一式（9ノード＋配線＋グループ枠）をキャンバスに生成する。Button接続を推奨。
SlimMigrate任意。既存の「Make2D実行」ノードを、端子を絞った軽量版「Make2D実行(スリム)」へ自動移行する（配線は端子名一致で自動移設、旧ノードは無効化して残す）。使わなくてよい上級者向けボタン。
出力ポート（1）
端子説明
Report構築の結果ログ。何個のノードを生成し、どの配線を張った／スキップしたかのサマリを返す。うまく組めたかはまずここで確認。

再実行しても安全な設計です。同じニックネームのノードは再生成せず再利用し、既存の配線はスキップするので、途中で切れた配線の修復にも使えます。

3使い方

  1. Grasshopperを開くRhino 8 でGrasshopperを起動する（このシステムはRhino 8 / Python 3 専用）。Rhinoエイリアス M2D からの起動にも対応。
  2. ビルダーノードを置くPython 3 Script コンポーネントを1つ置き、make2d_system\GH_SystemBuilder.py の内容を貼り付け、入力 Build（bool）と出力 Report を作る。
  3. Buildを押すBuild に Button を接続して押す。キャンバスにパイプライン一式（9ノード・全配線・グループ枠）が生成され、Rhino画面に操作パネルが現れる。
  4. パネルから順に実行ポップアップした操作パネルの「① 通り芯 → ② 断面プラン → ③ Make2D実行 → ④ 記号・注記」を上から順に実行していく。キャンバスを直接触らなくても回せる。
  5. コード修正は正本だけノードの挙動を直したくなったら、貼り直しは不要。C:\Python1\図面整理\ の正本.pyを編集すれば全ノードに反映される。
  6. 配線が切れたら再Build誤って配線を消しても、もう一度 Build を押せば足りない配線だけ張り直される（既存は温存）。

5サンプルモデルでデモ

🧩 サンプルモデル\01_柱梁フレーム.3dm

2×1スパンの柱梁フレーム＋通り芯線＋ガセット／浮きピース入り。パイプライン一式を試すデモ用モデル。

  1. モデルを開くサンプルモデル\01_柱梁フレーム.3dm を開く（またはワークセッションで参照）。
  2. Grasshopperで一括構築GHにビルダーノードを置いて Build を押す。9ノードと配線が生成され、操作パネルが立ち上がる。
  3. パネルから通り芯を拾うパネルの手順①で、フレームに付いている通り芯線から交点（X1-Y1…）を抽出する。
  4. 断面→Make2Dまで回す手順②③と進め、床伏図と断面図を1面ずつ試作。練習では書き戻しはオフのまま出力だけ確認すると安心。

6注意点・既知の罠

  • ⚠ Rhino 8（Python 3）専用

    このシステムは Rhino 8 の Python 3 コンポーネントで動きます。共有エンジン ghkit（C:\Python1\gh_ai_nodegen\ghkit.py）とスタブ置き場（make2d_system\stubs\）が前提です。Rhino 7 / IronPython 環境では動きません。

  • ⚠ スタブ方式＝正本の場所を動かさない

    生成されるノードは正本.pyを読み込むスタブです。正本（図面整理\ 内の各.py）やstubsフォルダを移動・リネームすると全ノードが動かなくなります。共有時はフォルダ構成をそのまま保ってください。

  • ℹ 旧配布形態「Make2D共有版.zip」との違い

    以前はGHファイル（260703作図メイク2D.gh）とローダースタブ群を固めた Make2D共有版.zip という配布物がありました。この make2d_system は 2026-07-03 の「システム化」以降の正本の配布形態で、両者は別物です。新規に使うならこちら（ワンボタン構築）が本命です。

  • ⚠ SlimMigrateは任意・移行後は目視確認

    SlimMigrate による軽量版への移行は任意です。旧・多端子ノードもそのまま全機能動作します。移行すると旧ノードは無効化されて残るので、問題ないことを目視確認してから手動で削除してください。
