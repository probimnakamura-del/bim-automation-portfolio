図面化パイプライン (Make2D)

接触検出ファミリー touch_finder_v2 / ref_touch_finder_v2

この部材に触れているモノを、まわりから自動でぜんぶ拾う——ガセットもブレースも取りこぼさない。

稼働中 通常版 v2 ／ 参照版 v2.7 GHノード ＋ ピッカー

MOTION PREVIEW

動作デモ

PREVIEW⟲ LOOP 10s

基準の本体を選ぶ → 触れている周辺部材（ガセット・ブレース・ブラケット）を自動で点灯 → 遠く離れた飛び地パーツだけ除外、という流れを10秒でループ再生しています。

1なにをするツール？

柱や梁の本体を1つ選ぶと、それに触れている周辺の部材（ガセットプレート・ブレース・ブラケットなど）を自動でぜんぶ見つけ出すツールです。図面化のときに「本体だけでなく、くっついている金物も一緒に描きたい」——その面倒な拾い出しを機械にやらせます。

3つの部品がセットになっています。touch_finder_v2（通常のモデル用）と ref_touch_finder_v2（ワークセッションで読み込んだ参照モデル用）が自動スキャンの本体で、touch_picker は「これは追加」「これは除外」を手でつまむGUID蓄積ピッカーです。参照版は点や薄い面を除外し、天端の高さで2段のサイズフィルタをかけ、さらに遠く離れた飛び地パーツ（同じアセンブリ定義に紛れ込んだ吊りピース等）を自動で除外します。

※ ワークセッション＝別の.3dmを「参照」として読み込む仕組み。参照モデルの部材は直接は編集できないため専用の拾い方が要ります。／ 拾った結果の最終的な足し引きは、下流の GH_FinalSelect に集約する設計です。

通常のモデルでは touch_finder_v2（入力 InstIds / ExcludeKw / Tolerance、出力 TouchIds・TouchGeos・Count）を自動スキャンに使います。旧版のモーダル方式は廃止済みです。

3使い方

  1. 本体を渡すIFC属性検索 などで仕分けた本体（アセンブリ）のGUIDを InstIds へ。参照モデルなら ref版を、通常モデルなら touch_finder_v2 を使う。
  2. 許容距離を決めるTolerance で「どれだけ近ければ接触とみなすか」を設定。Run でスキャン開始。
  3. 拾いすぎ・飛び地を絞るノイズは ExcludeKw、大きすぎる部材は MaxSize、遠く離れた飛び地は FarTol で切り捨てる。
  4. 取りこぼしは手で足す自動で拾えなかった部材は touch_picker（追加用）でクリック蓄積。要らないものは除外用ピッカーへ。
  5. FinalSelectへ集約する自動結果 RefIds と手動の追加／除外を GH_FinalSelect の各端子へ。足し引きはあちらで確定する。

5サンプルモデルでデモ

🧩 サンプルモデル\01_柱梁フレーム.3dm

2×1スパンの柱梁フレーム＋通り芯線＋ガセット／浮きピース入り。接触検出のデモにそのまま使えます。

  1. モデルを開くサンプルモデル\01_柱梁フレーム.3dm を開く（参照で読むなら ref版を使用）。
  2. 基準の柱を渡す中央スパンの柱1本のGUIDを InstIds へ。Run でスキャン。
  3. 接触パーツが点灯その柱に触れるガセットやブレースがハイライトされる。混入している浮きピースは FarTol を効かせると消える。
  4. FinalSelectで確認結果を GH_FinalSelect に流し、確定セットの数を Count で確認する。

6注意点・既知の罠

  • ⚠ 手動の追加・除外は端子で分離する（フリーズ回避）

    GHのsolve中にモーダルなピック＋表示切替を行うとフリーズします。そのため自動スキャンは純計算だけに専念させ、手でのピックは touch_picker を別ノードとして立て、AddIds／DelIds 端子に分けて渡す設計です。1ノードに詰め込まないでください。

  • ⚠ 参照モデルのピックは ReferenceObjectSelect が必須

    ワークセッションの参照モデルは通常の選択では拾えません。参照版は専用の参照オブジェクト選択を使います。参照で読み込んでいるのに拾えないときは、モデルが参照として正しく入っているかを先に確認してください。

  • ⚠ 飛び地パーツの混入は FarTol で切る

    アセンブリ定義に遠く離れた吊りピース等が紛れ込んでいると、接触判定が巻き込んでしまいます（例: 柱定義に十数m離れたピースが混入）。対角最大のアンカーから一定距離を超えるものを FarTol で除外してください。

  • ⚠ 自動結果は保持される・実機での最終確認を

    自動スキャン結果はキャッシュに保持され、モデルを切り替えても消えません。意図せず前のセットが残ることがあるため、重要案件では最終セットを目視で確認してください（ref版の一部は実機未検証項目あり）。
