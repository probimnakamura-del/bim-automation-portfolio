ライン整列 GH_AlignEndToLine

基準ラインをクリックで拾うだけ。バラバラな部材の端がスッと面一にそろう。

稼働中 Grasshopper Pythonノード Run立上りで実移動

MOTION PREVIEW

動作デモ

PREVIEW⟲ LOOP 10s

端がバラバラの部材群 → 基準ラインをクリックで拾う → それぞれの端がラインへ平行移動して面一にそろう、という流れを10秒ループで再生しています。

1なにをするツール？

選んだ部材の端を1本の基準ラインへ、まとめて面一（つらいち）にそろえるノードです。基準ラインは Pick ボタンでRhino上のライン・エッジ・ブロック・ソリッドをクリックして拾うだけ。あとは部材群がライン直交方向へ平行移動して、端がラインの位置にピタッとそろいます。ワークセッションの参照モデルにある線やエッジも基準として拾えます。

Offset で「ラインからどれだけ出す／引くか」を符号付きで指定できます。0で面一、プラスでラインを越えて出す（アウトセット）、マイナスで手前に止める（インセット）。姉妹ツールの GH_DropOnRef（参照天端への0タッチ）と同じ「端を基準へ突き合わせる」発想を、こちらはライン基準にしたオフセット特化版です。操作感（Preview→Run立上り→Ctrl+Zで取消）も共通です。

※ 面一＝複数部材の端を同じ面にそろえること。／ ライン直交方向＝基準ラインに対して垂直な水平方向。部材はこの向きに平行移動します。

3使い方

  1. 部材を渡す端をそろえたい部材のGUIDを Ids に接続。
  2. 基準ラインを拾うPick ボタンを押し、Rhino上の基準にしたいライン／エッジをクリック。参照モデルの線でもOK。直接渡すなら Line 端子に曲線を接続（そちらが優先）。
  3. 出し引きを決めるOffset で面一(0)／アウト(＋)／イン(−)を指定。逆側の端を基準にしたいときは FarEnd をオン。
  4. プレビューで確認Preview をオンにすると、緑ゴースト＋橙の基準ライン＋移動後の端位置と移動量が表示される。方向が90°ずれて見えたら Rotate90 で補正。
  5. Runで実行Run を押して離す（立上りエッジ）と実際に平行移動。ずれても Ctrl+Z で取消可。冪等なので適用後の移動量は≒0。

4つながり（ワークフロー）

単独で使えます。 他のノードにつながず単体で完結する整列補助ツールです。参照モデルの天端へ底面をそろえたいときは、同じ操作感の姉妹ツール GH_DropOnRef（0タッチ着地）が使えます。

5サンプルモデルでデモ

🧩 サンプルモデル\04_落下整列.3dm

梁の上に浮いた部材＋基準ライン。GH_DropOnRef着地とGH_AlignEndToLine整列のデモ用。

  1. モデルを開くサンプルモデル\04_落下整列.3dm を開く。端がそろっていない部材群と基準ラインが入っている。
  2. 部材を選ぶ端をそろえたい部材のGUIDを Ids に渡す。
  3. ラインを拾うPick でモデル内の基準ラインをクリック。Preview で端がそろう位置を確認。
  4. Offsetを変えてみるOffset を＋／−に振ると、アウトセット／インセットの違いがゴーストで分かる。Run で確定。

6注意点・既知の罠

  • ⚠ Runは「立上りエッジ」でのみ実移動。まずPreview

    実際の移動は Run が False→True になった瞬間だけ起きます（Button推奨）。冪等設計なので適用後の移動量は≒0。押す前に Preview の緑ゴーストと橙の基準ラインで仕上がりを確認してください。ずれても Ctrl+Z で戻せます。

  • ⚠ 移動方向が90°ずれて見えたら Rotate90

    部材はライン直交方向へ平行移動します。短いエッジなどを拾って向きが90°違って見えるときは Rotate90 をオンにすると、ライン沿い方向へ切り替わって補正できます。

  • ⚠ 基準にする端は「近い端」が既定

    そろえる基準は、ラインに近いほうの端が既定です。部材の反対側の端をラインに合わせたいときは FarEnd をオンにしてください。

  • ⚠ Line端子を接続するとPickより優先

    Line 端子に曲線を接続すると、そちらが基準として優先されます。クリックで拾った線を使いたいときは Line 端子を空にしておいてください。
