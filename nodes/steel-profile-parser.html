鉄骨モデリング支援

JIS鋼材断面パーサ steel_profile_parser

断面名の文字列を渡すだけで、正しい3D断面が生まれる——鉄骨モデリングの土台となる部品。

稼働中 共有ライブラリ Rhino8 Python3

動作デモ

PREVIEW⟲ LOOP 10s

「H-400×200×8×13」のような断面名 → 種別を判定して寸法へ分解 → その断面の3Dメッシュを生成、という流れを10秒でループ再生しています。

1なにをするツール？

鉄骨の断面は「H-400×200×8×13」「BOX-500×500×19」「φ-267.4×8.0」のように、記号＋寸法の文字列で呼びます。このツールは、その断面呼称の文字列を読み取り、正しい断面形状の3Dメッシュ（立体）に変換する縁の下のパーサ（解析器）です。

JISや現場でよく使われる表記ゆらぎ（H / BH / HW / HM / HN、□ / ■、[ / C、φ / PIPE / STK など）を吸収して、H形・角形鋼管・円形鋼管・みぞ形・山形・カットティー・鋼板の7種に振り分け、H・B・板厚などの寸法に分解します。断面積の概算も出せます。

※ 単体で図面を描くツールではなく、鉄骨モデリング系ツールや各種インポータが内部で呼び出す「部品」です。「断面名さえ渡せば形になる」土台を担います。

2入出力（渡すもの・返るもの）

GHの端子ではなく、コードから呼び出す関数群です。断面名の文字列を渡すと、寸法・断面積・3Dメッシュが返ります。

3使い方

  1. 断面呼称を用意するモデル化したい部材の断面名を文字列で用意（例 "H-400x200x8x13"）。IFCやTeklaの断面名をそのまま渡せます。
  2. 種別を自動判定接頭辞のゆらぎ（BH / HW / □ / [ など）を吸収して、H/BOX/PIPE/C/L/T/PLのどれかに振り分けます。
  3. 寸法へ分解する{type:'H', H, B, tw, tf} のような寸法辞書に分解。想定外の書式なら None を返すので、呼び出し側でチェックします。
  4. 断面ループ→押し出し2Dの断面ポリゴンを生成し、部材長ぶん押し出して3Dメッシュ（rhino3dm）にします。
  5. モデルへ配置生成したメッシュを他ツールがRhino/GHへ配置。断面積が要れば section_area で概算も取得できます。

4つながり（ワークフロー）

インポータ等が断面名を形にするために内部で呼び出しますが、断面名を持つあらゆる場面で単体のライブラリとしても使えます。

5実データでの使いどころ

🔩 鉄骨モデルの断面文字列 → 3D形状

IFCやTeklaから拾った断面名を、モデルや数量に落とし込む場面で使います（専用のサンプル.3dmは不要）。

  1. 断面名を集めるインポータが読み取った部材の断面名文字列（H-…/BOX-…/φ-…）をそのまま本パーサへ渡します。
  2. 形状に変換返った寸法・3DメッシュをそのままRhino/GHへ配置。表記ゆらぎは自動で吸収されます。
  3. 数量にも流用断面積（mm²）を section_area で取り出し、重量・数量集計に回せます。

6注意点・既知の罠

  • ⚠ Python3専用ライブラリ。クラシックGHでは動かない

    型ヒント（typing）を使うPython3モジュールです。IronPython2.7のクラシックGHコンポーネントでは全停止します。Rhino8のPython3ノードやvenvから使ってください。

  • ⚠ 未対応の書式は None が返る

    全角/半角の混在や想定外の表記は解析できず None になります。呼び出し側で必ずNoneチェックをして、未対応の断面名を弾いてください。

  • ⚠ 断面積はフィレット無視の近似

    R部（フィレット）を無視した概算です。ざっくりした重量・数量には十分ですが、厳密な断面性能が要るときはカタログ値を使ってください。
