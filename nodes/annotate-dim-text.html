Rhinoコマンド・配布ツール

注記ツール annotate_dim_text

クリックだけで、寸法と文字注記を次々と連続作図——GH不要のRhinoコマンド。

稼働中（チーム配布済） 配布zipあり / IFC方式 Rhino 8（IronPython2.7 / CPython3両対応）

MOTION PREVIEW

動作デモ

PREVIEW⟲ LOOP 10s

2点をクリックすると寸法線が伸びて数値ラベルが現れ、連続モードなら点を打つたびに寸法が次々連結。文字注記も同じ操作感で置ける様子を10秒でループ再生しています。

1なにをするツール？

Rhinoのコマンドラインから起動するGH不要の注記ツールです。2点をクリックするだけで寸法線を引き、連続モードなら点を打つたびに寸法が次々つながります。数値は Enterで実測値そのまま、数字を打てば表示だけ上書きも可能。文字（文言）の配置、既存寸法の位置調整・分割、他オブジェクトからの見た目コピーまで、図面の注記まわりを1本でこなします。

起動は Rhinoで 注記 または ANNO と打つだけ。AnnotateTools.rui の9個のアイコンを入れておけば、「寸法」「文字」「記号」「編集」「位置調整」「分割」「コピー適応」「設定」など、使いたい機能へワンクリックで直行できます。よく使う設定はプリセットに名前を付けて保存・呼び出しできます。

※ 寸法の向きは「自動／整列／水平／垂直／ライン」から選べ、「自動」は水平・垂直を即決し、斜め2点なら周辺の通り芯・部材から角度を検出して提案します。／ 端部マークは既定で円。専用の寸法スタイル ADT_Annotate を使うので、既存の寸法設定を壊しません。

3使い方

  1. 起動するRhinoのコマンド欄で 注記 か ANNO と入力（またはツールバーの AnnotateTools アイコンをクリック）。
  2. 機能を選ぶメニューから「寸法」「文字」などを選択。ツールバーの各ボタンからならメニューを飛ばして直行できる。
  3. 寸法を打つ始点→終点を2点クリック。値は Enter で実測、数字を打てば表示上書き。連続寸法ONなら点を打つほど寸法が連結。
  4. 向き・オフセットを調整始点ピック中に 向き / 連続 / オフセット をクリックで切替。斜め線は「自動」が周辺の通り芯・部材角度を提案。
  5. 文字を置く「文字」で文言を入力し1点で配置。フォント・色・高さは設定に従う（下線も可）。
  6. 整えるずれた寸法は「位置調整」で再オフセット、「コピー適応」で既存注記の見た目を取り込み、「プリセット」でよく使う設定を保存・呼出。

4つながり（ワークフロー）

単独で使えます。 GHパイプラインに依存しない独立コマンドで、どんな2D図面にも後から注記を足せます。Make2Dで起こした図面や既存の詳細図の仕上げ（寸法入れ・部材名記入）に手作業で使うのに向いています。GHノード版として GH_AnnotateDim が別に存在します（実行方式は本ツールの流儀を踏襲）。

5サンプルモデルでデモ

📐 サンプルモデル\02_2D図面.3dm

通り芯記号（X1,X2,Y1…）・断面符号（A-A）・タイトル文字入りの2D図面。通り芯抽出・断面符号抽出・統合JSON・注記ツールのデモ用。

  1. 図面を開くサンプルモデル\02_2D図面.3dm をRhinoで開く。
  2. ツールを起動コマンド欄で 注記 と入力し「寸法」を選ぶ。
  3. 通り芯間を寸法化X1→X2の通り芯を2点クリック。連続モードで X2→Y1… と打てば芯々寸法が一気に並ぶ。「自動」向きが通り芯の角度を拾う。
  4. 部材名を注記「文字」で「H-400×200」などの部材名を配置し、下線オプションで見出し風に仕上げる。

6注意点・配布のしくみ

  • 📦 配布zipは「解凍したフォルダが本体」（IFC方式）

    チーム配布用zip（注記ツール配布\ でビルド）は、解凍先フォルダが自分の場所を自動認識するため、社内でパスを揃える必要がありません。どこに解凍しても動きます。付属の AnnotateTools.rui を組み込むと9個のアイコンが並び、専用寸法スタイル ADT_Annotate も用意されます。

  • ⚠ 設定はAPPDATA、共有プリセットはフォルダ直下

    個人の設定・プリセットは %APPDATA%\AnnotateTools に保存されます（ツールの置き場所に依存しない）。一方、ツールフォルダ直下の annotate_dim_text_presets.json は共有プリセットとして合成読込されるので、ここに入れて配布すればチーム標準設定を配れます。

  • ⚠ 既存の寸法スタイルは壊さない設計

    本ツールは専用スタイル ADT_Annotate を使い、既定表示（小数第一位まで・端部＝円マーク）を起動時にそのスタイルへ適用します。他の寸法スタイルには触らないので安心して使えますが、見た目を揃えたいときは「寸法に反映」で現行スタイルへ設定を書き込めます。

  • ⚠ IronPython2.7 / CPython3 両対応を維持

    Rhino 7（IronPython2.7）でも動くよう両対応で書かれています。改修する場合は f-string や := などPy3専用構文を持ち込まないでください（IronPython側で全停止します）。
