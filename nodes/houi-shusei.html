1なにをするツール？

図面（DXF / DWG / PDF）の中にある「東・西・南・北」の方位を示す1文字だけを、まとめて別の方位へ書き換えるツールです。たとえば「東」を全部「西」に、「南」を「北」に——といった方位反転を、フォルダ内の図面に対して一括でかけられます。

いちばんの特徴は“サージカル（外科手術的）置換”。原点の近く（PDFの場合は用紙の左下エリア）にある単独の1文字だけを対象にし、それ以外の文字・線・図形には一切手を触れません。だから「東立面図」のような熟語の中の「東」は書き換わりません。誤爆が命取りになる図面作業でも安心して使えるよう、置換後は元データと自動で差分照合し、「予定外の変更が起きていないか」のチェック結果と、変更箇所をマーキングした確認PDFを自動で出力します。

※ 方位反転が必要になる場面＝図面の向き（見上げ／見下ろし・鏡像配置）が変わったときなど。／ サージカル置換＝狙った1文字だけをバイト単位で差し替え、ファイル全体は極力そのまま保つやり方。

3使い方

  1. ツールを起動する配布された 方位修正ツール.exe を起動する（EXE版なのでPython環境は不要）。
  2. 図面を選ぶ方位を直したい図面フォルダ、または個別のDXF / DWG / PDFを選択する。
  3. 変換ルールを決める「東→西」「南→北」など、変えたい方位の組み合わせにチェックを入れる。
  4. まずプレビューで下見実行モードを下見にして走らせ、どの1文字が検出されたかを確認資料PDFで確認。狙いがずれていれば原点近傍の半径を調整。
  5. 本実行する問題なければ本実行。修正済みファイルとレポート、確認PDFが出力される。
  6. 予定外変更チェックを見る最後に「予定外の変更 0 件」になっているかを必ず確認。方位以外が動いていないことを担保する安全弁。

4つながり（ワークフロー）

このツールは単独で使えます。図面ファイルを渡すだけで完結する、独立したユーティリティです（DWGを扱う際だけ、下記のODA File Converterが別途必要）。

5実際に試すには

🧭 手元のDXF / PDF図面で試すのが手軽

このツールは3Dモデル（.3dm）ではなく2Dの図面ファイルを扱うため、専用サンプルモデルはありません。方位記号の入ったDXFやPDFで試すのがおすすめです。

  1. テスト用にコピーを用意まず対象図面を別フォルダにコピーし、そのコピーで試す（原本はそのまま残す）。
  2. 下見モードで検出を確認プレビューで走らせ、確認PDFで「原点近傍の1文字だけ」に丸が付いているかチェック。
  3. 本実行して差分を確認本実行後、予定外変更チェックとマーキングPDFで、方位だけが変わったことを目視確認する。

6注意点・既知の罠

  • ⚠ DWGを扱うには ODA File Converter が必要

    DWGは直接編集できず、ODA File Converter を経由して変換します（EXEと同じ場所の oda_path.txt でパスを指定）。AutoCADのライセンス切れ環境やTrueViewでは保存不可のため、DWGを直したい場合はODAの導入を確認してください。DXF / PDFはそのまま扱えます。

  • ⚠ 対象は「原点近傍・単独1文字」だけ

    安全のため、原点の近くにある単独の1文字だけを書き換えます。「東立面図」のような熟語内の「東」は意図的に無視されます（誤爆防止）。逆に、原点から離れた場所や複数文字と繋がった方位記号は拾われないので、検出されなかったときはまず配置と原点近傍半径を疑ってください。

  • ⚠ PDFは左下エリアの最近傍1件のみ

    PDFの場合は用紙の左下エリアで最も原点に近い1件だけを対象にします。1枚のPDFに複数の方位記号があるレイアウトでは取りこぼしが出るので、確認PDFで検出箇所を必ずチェックしてください。

  • ⚠ 本実行の前に原本のバックアップを

    予定外変更チェックという安全弁はありますが、書き換えは元ファイルに対して行われます。大事な納品図は実行前に別フォルダへ原本をコピーし、下見モードで検出を確認してから本実行するのが安全です。
