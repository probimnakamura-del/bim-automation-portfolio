図面化パイプライン (Make2D)

部材記号システム GH_PartMark / GH_PartsList

同じ形の部材にはいつも同じ記号を。採番も注記も部材表もまとめて自動。

稼働中 v2 ／ 統合版 GH_PartMarkAll make2d_system ⑥採用

動作デモ

PREVIEW⟲ LOOP 10s

部材群を見て → 形の同じもの（同一の材質×寸法）をまとめて同じ記号を自動採番 → 2段注記を付けて部材表に員数集計、という流れを10秒でループ再生しています。

1なにをするツール？

製作図に欠かせない部材の仮記号（符号）付けと、部材表づくりを丸ごと自動化するシステムです。モデルの中の部材を1つずつ見て、同じ形（同じ材質×同じ寸法）のものには必ず同じ記号を振る——だから「同じ梁なのに別番号」「別の梁なのに同じ番号」といったミスが起きません。

付ける注記は2段構成（上段に寸法／断面、下段に「＜記号＞」＋下線＋引出線）で、レイアウトも自動。あわせて部材表（材質・員数・長さ・面積・重量）も生成します。心臓部の partmark_core.py が「形状サイン式・断面検索・採番表」の唯一の真実を持ち、記号付けの GH_PartMark と一覧表の GH_PartsList がそれを共有します。GH_PartMarkAll は両者を束ねた統合版で、make2d_system の⑥ノードとして正式採用されています。

※ 仮記号＝製作図で部材を呼ぶための符号（例: ★-00001）。／ 形状サイン＝面数・寸法・材質などから作る“形の指紋”で、同じ指紋なら同じ記号にします。／ 採番表はログ用の.3dmのユーザーテキストに保存され、次回以降も番号がぶつかりません。

3使い方

  1. 対象部材を渡す記号を付けたい部材の Ids（と形状）を接続。材質・断面を属性から読むためGUIDが要る。GH_FinalSelect の確定セットが便利。
  2. Make2Dの配置情報をつなぐMake2D実行の SecXf / SecBox / SecNames を渡すと、図面上の正しい位置に注記が並ぶ。
  3. 採番ルールを決めるPrefix（記号の頭）・Round（寸法の丸め）・SectionKey を設定して Run。
  4. 記号と部材表を確認する同形状に同じ記号が振られ、Table に員数・重量が集計される。PL（プレート）判定などは確認ポップアップで承認。
  5. 図面へ書き込む・書き出すよければ Write / Bake で注記を図面へ、Export で部材表を書き出す。採番表はログ.3dmに保存され次回も継続。

5サンプルモデルでデモ

🧩 サンプルモデル\01_柱梁フレーム.3dm

2×1スパンの柱梁フレーム＋通り芯線＋ガセット／浮きピース入り。同形状の梁が複数あり、採番と員数集計のデモに向きます。

  1. モデルを開くサンプルモデル\01_柱梁フレーム.3dm を開く。
  2. 部材を渡す柱・梁のアセンブリを Ids へ。Run で採番。
  3. 同形状に同記号を確認同じ寸法の梁が同じ記号になり、Table の員数がその本数と一致するのを確認。
  4. 書込は練習ではオフ練習では Write / Bake を押さず、Table と注記プレビューだけ眺めると安全。

6注意点・既知の罠

  • ⚠ 材質・断面は属性から読む（Ids入力が前提）

    記号は「材質×寸法」で決まるため、材質や断面の情報が部材の属性に入っている必要があります。形状（Geos）だけでなく Ids を渡してください。属性が無いと既定の MatDefault にフォールバックします。

  • ⚠ 採番表はログ.3dmに永続化される

    採番表（PartMarkRegistry_v1）はログ用の.3dmのユーザーテキストに保存され、次回以降も番号が引き継がれます。番号を振り直したいときは Reset を使ってください。ログ.3dmを消すと採番の連続性が失われます。

  • ⚠ 毎solveの全再計算は署名キャッシュで抑制済み

    入力の署名でキャッシュし、変化が無ければ再計算しません。最新の状態に強制更新したいときは Run を立ち上げてください（Runが強制リフレッシュのトリガ）。

  • ⚠ PL（プレート）判定は確認ポップアップで承認

    プレート系の部材は判定が分かれやすいため、書き込み前に確認ポップアップが出ます。内容を見て承認してから Write / Bake を実行してください。
