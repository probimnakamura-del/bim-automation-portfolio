BoxDriveSync Box→スプレッドシート同期

Boxに積まれた指示書を、ファイル名から読み取ってスプレッドシートへ自動転記する——手入力ゼロの事務ロボット。

稼働中と推定（1日3回 自動実行） タスクスケジューラ常駐 Rhino/GH とは独立

MOTION PREVIEW

動作デモ

PREVIEW⟲ LOOP 10s

Boxフォルダに増えた指示書のファイル名を走査し、FID・資料名・版を抜き出してスプレッドシートへ差分転記、GASで双方向同期するまでを10秒でループ再生しています。

1なにをするツール？

Rhino/Grasshopper とは関係のない、純粋な事務作業の自動化ツールです。ある実案件（大型新築工事・付帯鉄骨関連）のBox Drive の指示書フォルダを定期的に見に行き、増えた・変わったファイルをGoogleスプレッドシートへ自動で書き込みます。「フォルダの中身を目視で台帳に転記する」あの地味な作業を、まるごと肩代わりします。

やっていることは3つ。フォルダを再帰スキャンしてファイル一覧を取り、ファイル名から FID・資料名・バージョンなどの情報を切り出し、スプレッドシートと突き合わせて差分だけ同期します。仕上げに GAS（Google Apps Script）のWebhookを叩き、スプレッドシート側でのシート一括更新・双方向同期を自動で発火させます。タスクスケジューラで1日3回、勝手に走る想定です。

※ FID＝資料を管理する番号のこと。／ GAS（Google Apps Script）＝Googleスプレッドシート側で動く自動化スクリプト。／ 再帰スキャン＝サブフォルダの奥まで全部たどって調べること。

2入口と出口

読み取るもの（入力）
対象説明
入Box フォルダBOX_ROOT_PATH 配下のファイル一覧（再帰スキャン）。ファイル名からFID・資料名・版を抽出。
入既存シートGoogleスプレッドシートの現在値。差分（新規・更新）を判定するための突き合わせ元。
書き込むもの（出力）
対象説明
出スプレッドシート差分だけを転記（新規行の追加・既存行の更新）。
出GAS Webhook同期後にGoogle側スクリプトを呼び出し、シート一括更新・双方向同期を発火。

3使い方

  1. 認証情報を置くGoogle Cloud のサービスアカウント鍵 credentials.json を BoxDriveSync\ に配置。これが gspread 認証に必須。
  2. 対象を設定するbox_to_sheets.py の BOX_ROOT_PATH（見に行くBoxフォルダ）／SPREADSHEET_KEY（書き込む先）／WORKSHEET_NAME を対象案件に合わせる。既定は特定案件の付帯鉄骨フォルダ。
  3. 実行する（または自動化）手動なら box_to_sheets.py を実行。常駐させるなら register_task.ps1 でタスクスケジューラに1日3回登録する。
  4. 同期を確認するスプレッドシートに新規・更新行が反映される。ファイル名から抽出したFID・資料名・版が列に入る。
  5. Google側の連携が走る同期の最後に GAS_WEBHOOK_URL が叩かれ、Google側の BoxDriveSync_GAS.js がシート一括更新・双方向同期を実行する。

4つながり（ワークフロー）

単独で使えます。 Rhino/Grasshopper のツール群とは独立した事務ツールで、他のノードとの入出力連携はありません（BIM_Shield や BIM Dashboard とも連携なし）。相方はローカルのBox Driveと、Google側にデプロイ済みの GAS スクリプトだけです。

5注意点・既知の罠

  • ⚠ 特定案件がハードコードされている

    BOX_ROOT_PATH は特定案件（大型新築工事・付帯鉄骨）の実パスを直接指しています。汎用化されていないので、他案件に使うには BOX_ROOT_PATH / SPREADSHEET_KEY / WORKSHEET_NAME の書き換えが必要です。

  • ⚠ GAS本体はGoogle側にデプロイ済み。jsを直しても効かない

    リポジトリ内の BoxDriveSync_GAS.js はソース保管用のコピーです。実体はスプレッドシートにバインドされてGoogle側にデプロイ済みなので、このファイルを編集しても、Googleでデプロイし直さない限りWebhook側の挙動は変わりません。

  • ⚠ credentials.json は秘密鍵。取り扱い注意

    credentials.json は Google Cloud のサービスアカウント鍵で、これが無いと認証に失敗して何も同期できません。逆に秘密情報なので、共有やバックアップの際は流出しないよう注意してください（一般公開リポジトリ等に入れない）。

  • ⚠ 実行ログが見当たらず、直近稼働は要確認

    タスクスケジューラに1日3回で登録されている構成は確認できましたが、フォルダ内に実行ログが見当たらず、直近で実際に動いているかは本調査だけでは確定できません。運用状況はご自身で最終確認してください。
