ブロック内サブ選択 subpick_block_multinode

ブロックの中の部品だけをクリックで選んで記録——複数ノードでも選択が混ざらないピッカー。

稼働中 GHパイソンコンポーネント Rhino7/8 両対応

MOTION PREVIEW

動作デモ

PREVIEW⟲ LOOP 10s

ブロックを指定 → ダイアログで内部部品にチェック → チェックした部品だけがアンバーで確定し、Srf端子から取り出せる、という流れを10秒でループ再生しています。

1なにをするツール？

Rhinoのブロック（＝複数部品をひとまとめにしたアセンブリ）の中から、欲しい部品だけをクリックで選び出して記録するピッカーです。ブロックは普通そのままだと「中の1部品だけ選ぶ」のが面倒ですが、このノードは専用のダイアログを開いて内部パーツを一覧チェックで選択できます。

選んだ部品は画面上でアンバーにハイライトされ、Srf 端子からそのジオメトリだけを取り出せます。かさ上げ鋼材の生成など「アセンブリの特定の面・部材を基準に何かを作る」下ごしらえに使います。

※ サブオブジェクト＝ブロックやポリサーフェスの「中の一部」。／ このノードの選択記録はブロックのIDと結びついて保持されるので、一度選べば再solveしても選択が残ります。

2入出力（GHノードの端子）

入力ポート（4）
端子説明
ID対象のブロックインスタンスのID（Guid）を入力。
showToggleを接続。Trueにすると選択ダイアログが開く。ここで内部部品をチェックする。
zoomToggleを接続。Trueにすると選択した部材へビューをズーム。
NodeID（任意）パネル等で "A" "B" 等の文字を入れると、強制的に別ノードとして記録を独立させる。同じブロックを複数ノードで扱うとき用。
出力ポート（3）
端子説明
Srfダイアログで選択された部材のジオメトリ。これが主役の出力。
Allブロックインスタンス内のすべての部材ジオメトリ。
Log選択履歴やエラーなどのログテキスト。うまく選べないときの確認用。

3使い方

  1. ノードを置いてブロックIDをつなぐGHにこのコンポーネントを置き、ID に対象ブロックインスタンスのGuidを渡す。
  2. ダイアログを開くshow のToggleをTrueにすると選択ダイアログが開く。内部部品の一覧が並ぶ。
  3. 部品にチェックする取り出したい部品にチェックを入れる。選んだ部品はビューでアンバーにハイライトされる。
  4. 結果を受け取るSrf に選択部材のジオメトリが出る。ビューを寄せたいときは zoom をオン。
  5. 複数ノードで使うときは NodeID を分ける同じブロックを別のノードでも扱う場合は、それぞれの NodeID に別々の文字を入れて記録を独立させる。これで選択が混ざらない。
🔧 派生ノード: subpick_l_angle_trim.py

同じ「ブロック内サブ選択」UIをベースにしたアングル（L形鋼）トリム用の派生版。選んだ部材から切断用のカッターソリッド（Cutters）と算出した部材幅（CalcBeamWidth）を追加で出力し、アングルの端部処理に使います。

5自分のモデルで試す

専用サンプルはありません。手元のアセンブリブロック（Tekla由来のブロックや、自作の複数部品ブロック）で試せます。

  1. ブロックを1つ用意複数部品からなるブロックインスタンスを1つ選ぶ。ifc2rhino で取り込んだアセンブリ（＝ブロック）が最適。
  2. IDを渡してダイアログを開くそのブロックのGuidを ID へ、show をTrueに。中の部品が一覧で出る。
  3. 1部品だけ選ぶ1つだけチェックして Srf を確認。狙った部品だけが取り出せることを体感できる。

6注意点・既知の罠

  • ⚠ このノードが生まれた理由＝「選択の混線」対策

    複数ノードで同じブロックを参照すると、Rhinoのサブオブジェクト選択が共有されて、あるノードの事前フィルタが別ノードに漏れる問題があります。本ノードはノード（InstanceGuidまたは手動NodeID）ごとに選択記録を独立させることでこれを回避しています。混ざったと感じたら NodeID を分けてください。

  • ⚠ Py3専用構文は避ける（Rhino7互換）

    本ノードはRhino7（IronPython 2.7）／Rhino8（Python3）の両対応です。改造する場合、f-string や := などPy3専用構文を入れるとRhino7側で全停止します。

  • ⚠ 選択後は解除しておく

    Rhinoのサブオブジェクト選択状態は、親オブジェクトが GetSelectedObjects に出ないなどAPI挙動にクセがあります。選択状態は保持せず「解除」で運用する設計です。ダイアログを閉じたら選択は記録側に残り、ビュー上の選択は解除されます。
