IFCスキーマ変換 IFC_Converter

古い規格のIFC2X3を、いまのIFC4へ。非対応の部品も自動でパッチして変換する。

稼働中 v1.0.0 CLI / GUI

MOTION PREVIEW

動作デモ

PREVIEW⟲ LOOP 10s

非対応エンティティを含むIFC2X3 → entity_patchesで事前置換し、Migratorでスキーマ移行、validateで検証 → IFC4と警告レポートを出力する流れを10秒でループ再生しています。

1なにをするツール？

古い規格「IFC2X3」で書かれたファイルを、いまの「IFC4」へ変換する専用ツールです。古いTeklaや他ソフトから出てきたIFCが2X3のままだと、最近のビューアや変換ツールでうまく扱えないことがあります。そこで、このツールでIFC4へ揃えておくと後工程が安定します。

変換エンジンには ifcopenshell の Migrator を使いますが、Migratorが対応していないIFC2X3固有の部品（例：IfcEquipmentElement など）がそのままだと変換に失敗します。本ツールは entity_patches.py で非対応エンティティを事前に互換のものへ置き換えてから変換するため、素のMigratorより通りやすいのが特長です。最後に ifcopenshell.validate で検証し、警告・エラーをレポートします。結合機能はなく、あくまでスキーマ変換だけを担います。

※ スキーマ＝IFCの「規格バージョン」。IFC2X3は2000年代の版、IFC4は現行版で使える部品や属性が増えている。／ Migrator＝ifcopenshellが持つ版間変換の仕組み。

3使い方

  1. 変換対象のIFC2X3を用意古い規格のIFC（バージョンが2X3のもの）を手元に置く。
  2. ツールを起動IFC_Converter\start.bat を実行（共有venv C:\Python1\.venv をアクティブ化して起動）。CLI版・GUI版どちらでも使える。
  3. 入力ファイルを指定変換したい input.ifc を選ぶ。
  4. 変換を実行① entity_patchesで非対応クラスを事前置換 → ② Migratorでスキーマ移行 → ③ validateで検証、が自動で走る。
  5. レポートを確認出力 output_IFC4.ifc と一緒に出る警告・エラーに目を通す。警告が多い場合は元データを見直す。
  6. 後工程へ渡すIFC4化したファイルは、結合（Join_IFC_model／Join_IF_model）や ifc2rhino の.3dm変換に流すと安定しやすい。

5実データでためす

🧩 手元のIFC2X3ファイル

専用のサンプル.3dmはありません。古い規格のIFC（2X3）を1つ用意して試すのがいちばん確実です。

  1. 規格を確認IFCファイル先頭付近の FILE_SCHEMA(('IFC2X3')) の記述で古い規格かを確認。
  2. 変換してみるstart.batから起動し、その2X3ファイルを input.ifc に指定して実行。
  3. 結果を見比べる出力の FILE_SCHEMA(('IFC4')) になっていること、レポートに致命的エラーが無いことを確認。

6注意点・既知の罠

  • ⚠ やるのは「スキーマ変換」だけ。結合はしない

    複数のIFCを1つにまとめる機能はありません。版を揃えるのが役割なので、結合したいときは変換後に Join_IFC_model や Join_IF_model を使ってください。

  • ⚠ 事前置換は「既知の非対応クラス」だけ

    entity_patches.py が置き換えるのは、あらかじめ登録された非対応エンティティです。未知の非対応クラスが混ざっていると変換で警告・エラーが出ることがあります。必ず検証レポートを確認してください。

  • ⚠ 共有venv（.venv）に依存

    ifcopenshellなどを含む共有仮想環境 C:\Python1\.venv を使います。この環境が壊れる／消えると起動できません。start.batが失敗するときはvenvの状態を先に確認してください。

  • ⚠ 変換後の検証は必ず目視

    版間変換は形状や属性の解釈が微妙に変わることがあります。validateの警告・エラーに必ず目を通し、重要なモデルは変換前後をビューアで見比べてください。
