使い方ガイドPPTX自動生成 _portfolio_build

コードを1回走らせるだけで、鉄骨自動化ツールの「使い方ガイド」スライドを自動で組み上げる資料ビルダー。

現役（出力ファイル実在） 補助ツール（資料生成） Node.js / pptxgenjs

MOTION PREVIEW

動作デモ

PREVIEW⟲ LOOP 10s

コードでのスライド定義 → node コマンドでビルド → PPTXファイルが生成される、という流れを10秒でループ再生しています。

1なにをするツール？

鉄骨製作図の自動化ツール群（部材カット・かさ上げ鋼材・部材記号など）を、人に説明するための「使い方ガイド」スライドをコードから自動生成する補助ツールです。PowerPointを手で1枚ずつ作る代わりに、スライドの中身をコードに書いておき、コマンド1回で .pptx を書き出します。

中身は Node.js の pptxgenjs（プレゼンをコードで作るライブラリ）を使った build_pptx.js 1本だけ。タイトル・配色・図解・説明文がスクリプト内に定義してあり、node build_pptx.js を走らせると プロジェクトのルート直下に「ポートフォリオ_使い方ガイド.pptx」が生成されます。資料を作り直したいときは、コードを直して再実行するだけで毎回同じ体裁のスライドが出来上がります。

※ BIM自動化パイプライン本体ではなく、その説明資料をつくるためのツールです。他のツールとファイルをやりとりする連携はありません。／ pptx＝PowerPointのファイル形式。

2操作と結果

GHノードではなくコマンドで動かすビルドスクリプトです。入力はコード内に埋め込まれているため、実質「1コマンドで1ファイルを出す」だけのシンプルな道具です。

3使い方

  1. フォルダへ移動ターミナルで _portfolio_build に移動する。
  2. 依存をそろえる（初回のみ）npm install を実行して pptxgenjs 等をインストール（node_modules が既にあればスキップ可）。
  3. 内容を編集する（必要なら）スライドのタイトル・配色・図解を変えたいときは build_pptx.js のスライド定義を書き換える。
  4. ビルドするnode build_pptx.js を実行。数秒で ポートフォリオ_使い方ガイド.pptx が生成される。
  5. できあがりを開くプロジェクトのルート直下に出た .pptx を PowerPoint で開いて確認。体裁を直したければ手順3〜4を繰り返す。

4つながり（ワークフロー）

使い方ガイドPPTX自動生成（このツール）

このツールは単独で使えます。他のノードとデータをやりとりする連携はなく、鉄骨自動化ツール群（部材カット・かさ上げ鋼材・部材記号など）を人に説明する資料をつくるための独立した補助ツールです。

5実際に動かすには

📊 専用サンプルモデルはありません（コード実行だけで完結）

3Dモデルを扱うツールではないため .3dm のサンプルはありません。スライドの中身はコードに書いてあるので、コマンドを1回走らせれば結果が確認できます。

  1. まずそのまま実行_portfolio_build フォルダで node build_pptx.js を実行し、既定のスライドが出ることを確かめる。
  2. 出力を開くプロジェクトのルート直下の ポートフォリオ_使い方ガイド.pptx を開いて中身を確認する。
  3. 1枚だけ変えてみるbuild_pptx.js のタイトル文字などを少し書き換えて再実行し、変更が反映されることを体感する。

6注意点・既知の罠

  • ⚠ Python ではなく Node.js のツール

    このカタログでは珍しく JavaScript（Node.js）製です。実行には Node.js が必要で、py や Python では動きません。初回は npm install で依存（pptxgenjs 等）をそろえてください。

  • ⚠ スライドの中身はコードにハードコード

    タイトルや図解は build_pptx.js の中に直書きされています。GUIで編集する仕組みはないため、内容を変えるにはコードを編集する必要があります。

  • ⚠ 出力先は プロジェクトのルート直下（上書きに注意）

    生成される ポートフォリオ_使い方ガイド.pptx は毎回同じ場所に同じ名前で書き出され上書きされます。手で編集した版があるときは、実行前に別名で退避してください。

  • ⚠ 本体パイプラインとは無関係の補助ツール

    BIM自動化の処理そのものには関与しません。あくまで説明資料を作るための道具なので、鉄骨ツールの動作確認とは切り離して考えてください。
