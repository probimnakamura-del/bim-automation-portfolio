動作デモ

PREVIEW⟲ LOOP 10s

Pythonコード（や日本語の指示）を ghkit エンジンが解析して、型が付いた「使えるGH Python3ノード」に組み立てるまでを10秒でループ再生しています。

1なにをするツール？

コードを書いても、それをGrasshopperで「使えるノード」にするのは地味に手間——入力端子を作り、型を設定し、標準の入力ソース（スライダやトグル）をつなぐ…。この一連を自動化する開発基盤です。手持ちの .py ファイルを渡すだけで、端子付きのGH Python3ノードが一気にキャンバスへ並びます。

心臓部は共有エンジン ghkit.py（1387行）。コードを3段階（AST解析／宣言コメント／AIによる補正）で読み解いて入出力端子を推論し、実際のGHノードとして組み立てます。実際にGHに貼って使うのは、この土台を呼ぶ薄いラッパー6種——全部入りの ghai_studio、コード→ノードの ghai_nodegen、「こういう処理がしたい」の日本語からコードを作る ghnl_codegen、モデル一覧の ghollama_models、ファイル選択の ghfile_picker、型ヒント修復の fix_ghdoc_hints です。

※ AI補正はローカルのOllama（PC内で動く無料のAI・既定 qwen2.5-coder）を使います。クラウドのAPIには一切つながず、完全オフラインで動くのが特徴です。／ AST＝コードの構造を機械が読める形に分解したもの。

3使い方

  1. コンポーネントを用意するGHキャンバスに ghai_nodegen（または全部入りの ghai_studio）を配置。コードフォルダから一括生成したいなら NodeGenランチャー を NODEGEN で起動しても良い。
  2. 材料を渡す手持ちの .py を Files へ（ghfile_picker で選ぶと楽）、またはコードを Code_Text へ。日本語で書きたいときは ghnl_codegen に指示文を入れてコードを作る。
  3. AI補正を選ぶ（任意）端子の型やリスト判定、要約をAIに整えさせたいなら Use_AI をオン。Ollamaを起動しておく（未起動なら補正なしで動く）。
  4. 生成するGenerate を押すと、端子付きのGH Python3ノードがキャンバスに現れる。Wire_Inputs をオンにしておけば、スライダ・トグル・パネルなどの入力ソースも自動でつながる。
  5. 型ヒントが戻ったら直す入力端子のヒントが「ghdoc Object」に勝手に戻ってしまったら、fix_ghdoc_hints の Run で全ノードを一括修復する。

この基盤は「使う」というより他ツールの土台です。NodeGenランチャー や section_lower のAI部品など、複数のツールが内部で ghkit を呼んでいます。だからこそフォルダ移動・改名は厳禁（下の注意点を参照）。

5注意点・既知の罠

  • ⚠ フォルダ移動・改名は複数システムを同時に壊す

    ghkit.py は多くのツールが絶対パスで参照しています（各ラッパーの先頭、make2d_system の SystemBuilder、section_lower のAI部品ほか）。C:\Python1\gh_ai_nodegen\ を移動・改名すると、それらが一斉にimportエラーで止まります。動かす場合は各参照側のパスを直す必要があります。

  • ⚠ AIはローカルOllama専用（クラウドAPIは使わない）

    AI補正はPC内のOllama（http://localhost:11434・既定 qwen2.5-coder）を直接叩きます。過去にあったClaude API版は廃止済みで、現行にAPI連携はありません。Use_AI を使うときはOllamaを起動してください（未起動でもAI無しで生成自体は動きます）。

  • ⚠ 型ヒントが「ghdoc Object」に勝手に戻る

    GHの仕様で、入力端子の型ヒントを設定し直した直後に「ghdoc Object」へ巻き戻ることがあります。エンジン側（materialize 内）で修復を入れていますが、それでも戻ってしまったら fix_ghdoc_hints で全ドキュメントを一括修復してください。

  • ⚠ RH8 Python3 専用。ランチャーはツールバー設定が要る

    これらは Rhino8 のPython3コンポーネント前提です（IronPythonでは動きません）。また NODEGEN ランチャーは rh_nodegen.py と NodeGenTools.rui を絶対パスで参照するため、移動時はセットアップスクリプトの再実行が必要です。
