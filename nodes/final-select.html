動作デモ

PREVIEW⟲ LOOP 10s

3種の選択（自動・追加・除外）を合流 → 和集合を作ってから除外を最優先で引く → 確定セットだけを画面表示・Rhino実選択・Make2D実行へ中継、という流れを10秒でループ再生しています。

1なにをするツール？

接触検出やピッカーなどあちこちのツールが出した「選択結果」を、1つの端子にまとめて最終セットを確定する中継ハブです。自動で拾ったもの・手で足したもの・手で外したもの——バラバラの選択を一箇所に集めて、「結局これが対象部材のすべて」と言い切れる状態を作ります。

合流のルールはシンプルです。最終セット =（自動結果 ∪ 手動追加）− 手動除外。除外（DelIds）が最優先で、GUID（部材の識別番号）が一致するものに加えて形状の一致でも二段構えで取り除きます。確定したセットは、画面上への強調表示・Rhinoでの実選択・そして下流の Make2D実行 へ、同時に中継されます。何がいくつ残ったかは Count で常に見えるので、選択の“流量”が一目で分かります。

※ ∪（和集合）＝重複を除いて足し合わせること。／ DelIds最優先＝「追加」と「除外」で同じ部材がぶつかったら、必ず除外が勝つ、という意味です。

3使い方

  1. 自動結果を土台に置く接触検出の TouchIds / RefIds（と形状）を Ids / Geos へつなぐ。
  2. 手動の足し引きをつなぐ追加ピッカーの出力を AddIds へ、除外ピッカーの出力を DelIds へ。touch_picker を2個立てて役割分担させる。
  3. 合流結果を確認するsolveすると確定セットが計算される。Count で点数、Highlight で画面の色を見て過不足をチェック。
  4. 実選択して見比べるSelect をオンにするとRhino上で実際に選択される。取りこぼしがあれば AddIds／DelIds を調整。
  5. Make2Dへ渡す確定した OutGeos を GH_Make2DExecutor の TouchGeo へ。背景の見え線として図面に描かれる。

5サンプルモデルでデモ

🧩 サンプルモデル\01_柱梁フレーム.3dm

2×1スパンの柱梁フレーム＋通り芯線＋ガセット／浮きピース入り。合流と足し引きの練習に使えます。

  1. モデルを開くサンプルモデル\01_柱梁フレーム.3dm を開く。
  2. 自動結果を流す接触検出で柱1本まわりのセットを作り、Ids / Geos へ接続。
  3. 1つ足して1つ外す追加ピッカーで隣のブレースを足し、除外ピッカーで余計な浮きピースを外す。AddIds／DelIds を効かせる。
  4. 除外が勝つのを確認わざと同じ部材を「追加」と「除外」の両方に入れてみる。Count が示すとおり除外側が勝つのを確認。

6注意点・既知の罠

  • ⚠ 除外（DelIds）は必ず最優先

    同じ部材が「追加」と「除外」の両方に入っていると、除外が勝ちます。GUID一致に加えて形状一致でも二段で除去するため、追加したはずのものが消えるときは除外側に紛れていないか確認してください。

  • ⚠ 選択の関所＝ここで流量を必ず見る

    このノードは複数ツールの結果が合流する要（かなめ）です。Count と Report で「いくつ入っていくつ残ったか」を必ず確認する習慣にすると、下流のMake2Dでの過不足を早く発見できます。

  • ⚠ Conduit（画面描画）の後始末

    強調表示はConduitで画面に描かれます。GHのダイアログを×で閉じたりコンポーネントを作り直すと描画が画面に残る（孤児化）ことがあります。表示が消えないときはノードを一度無効化→有効化してリセットしてください。
