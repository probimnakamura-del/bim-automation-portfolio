部材カット自動延長ツール v3

部材の端を自動でカットして、そのまま延長——鉄骨の突き合わせをワンノードで片付ける「Smart Member Processor」。

稼働中 v3系（現行・GrooveSide対応） Grasshopper Pythonノード

MOTION PREVIEW

動作デモ

PREVIEW⟲ LOOP 10s

端に隙間がある部材 → カッター形状で端を削ぐ → 自動延長で基準部材へ突き合わせる、という一連の流れを10秒でループ再生しています。

1なにをするツール？

鉄骨部材どうしの端部の取り合い（カット＆延長）を自動化するノードです。「基準になる部材（ID）」と「加工したい部材（A）」を選ぶと、基準の形にぴったり合うカッター形状を自動で作り、対象部材の端をブーリアンで削ぎ落としてから、基準面へ届くまで自動で延長します。手作業なら1本ずつやる「端を切って、伸ばして、突き合わせる」を一括で片付けます。

TrimMode で加工のやり方を4通りに切り替えられます。ふつうの梁柱は「通常カット＆延長（0）」、かさ上げ材やデッキ受けのような固定寸法スリットが要るものは「上フランジ溶接特殊カット（1）」、削るだけで伸ばさないなら「単純ブーリアン（2）」、複数の板を1枚とみなしてスリットを切るなら「PL強制統合（3）」。v3 では TrimMode1 のときに開先（斜めの溶接開口）がどちら側かを GrooveSide で手動指定できるようになりました。

※ カッター形状＝相手にめり込ませてブーリアン差で削り取るための立体。／ 開先＝溶接のために部材端を斜めに削った開口。／ クリアランス＝突き合わせに残す意図的な隙間。

3使い方

  1. ノードを置くGrasshopper に本ノードを配置。フォルダ直下の 部材カット自動延長ツール_v3.py が現行版（v1/v2は旧版として温存）。
  2. 基準と対象を渡す突き合わせ先の部材を ID へ、削って伸ばしたい部材を A へ接続（どちらもリスト可）。
  3. 加工方式を選ぶTrimMode を用途に合わせて設定（通常は 0）。かさ上げ材など特殊スリットは 1、削るだけなら 2、板の統合スリットは 3。
  4. 隙間と向きを調整Clearance で突き合わせ隙間、CutSide でどちらの端を加工するかを指定。TrimMode1なら GrooveSide で開先側を明示。
  5. 結果を確認するResult に延長済み部材が出る。うまく削れないときは Cutters と Log でカッター形状とログを確認。
  6. モデルへ反映仕上がったら Bake などでモデルに反映。上流のかさ上げ鋼材で生成した部材の端処理にそのまま使える。

5サンプルモデルでデモ

🧩 サンプルモデル\01_柱梁フレーム.3dm

2×1スパンの柱梁フレーム＋通り芯線＋ガセット／浮きピース入り。接触検出・Make2D・部材記号・かさ上げ・部材カットのデモ用。

  1. モデルを開くサンプルモデル\01_柱梁フレーム.3dm を開く。
  2. 基準と対象を選ぶ柱（または大梁）を ID、それに取り付く小梁・ピースを A に指定。
  3. 通常モードで実行TrimMode=0、Clearance=0 前後で solve。端が削られ、隙間が延長で閉じるのを確認。
  4. 特殊モードも試すかさ上げ材を対象にして TrimMode=1 にすると、固定寸法スリットの挙動を確認できる。

6注意点・既知の罠

  • ⚠ フォルダに v1 / v2 / v3 が同居。使うのは v3

    直下に3世代が並んでいます。現行は 部材カット自動延長ツール_v3.py（GrooveSide対応版）。v1/v2 はロールバック用に温存されているだけなので、間違って古い版をノードに読み込まないよう注意。

  • ⚠ TrimMode1 は固定寸法スリットの専用モード

    「上フランジ溶接特殊カット（1）」は、かさ上げ材・デッキ受け向けに 140／287・19／10mm逃げといった固定寸法のスリットを切る用途特化です。一般的な梁柱の突き合わせには通常カット（0）を使ってください。

  • ⚠ 複数ノードで同じブロックを参照するとサブ選択が漏れる

    ブロック内サブオブジェクト選択（show）は、複数ノードで同じブロックを参照するとRhino側の選択状態が共有され、事前フィルタが漏れることがあります。本ツールはノード／位置ごとに独立記録する設計ですが、同時に複数のピックUIを開かないのが安全です。

  • ⚠ Clearance は数値でも "C20" でも受け付ける

    クリアランスは 20 のような数値のほか、"C20" のような文字列でも指定できます。0タッチにしたいときは 0 を渡します。
