IFC 原点キューブ結合 Join_IFC_model

複数の IFC を原点キューブのズレから自動で位置合わせして、一発でピタッと合体——階ごとに補正する爆速版。

稼働中 v6.1（超高速フルバースト版） Python スクリプト（Tk GUI）

MOTION PREVIEW

動作デモ

PREVIEW⟲ LOOP 10s

基準 IFC の原点キューブを軸に、ズレた追加モデルの差分を自動計測 → GUI で確認 → 階ごとに補正して1つの IFC へ合体、という流れを10秒でループ再生しています。

1なにをするツール？

ばらばらに書き出された 複数の IFC ファイルを、1つに統合するツールです。難しいのは「各モデルの位置（座標原点）が揃っていない」こと。そこでこのツールは、各 IFC に仕込んだ「原点キューブ」の座標の差から、モデルごとのズレ（オフセット）を自動で計算し、ぴったり重ねてから結合します。

結合前には tkinter の GUI ダイアログで「どれだけズレているか」を数値で表示してくれるので、勝手にズレたまま合体してしまう事故を防げます。補正は 階（IfcBuildingStorey）単位で当て、各要素の相対座標はそのまま維持します。

v6.1 は 速度に全振りした「超高速フルバースト版」。重い numpy 行列計算や、マテリアルの重複チェック（reuse_identities）を省いて、とにかく速く合体させることを優先しています。基準1ファイル＋追加を複数まとめて処理できます。

※ 原点キューブ＝各モデルの基準位置を示すために置いておく小さな立方体。これの座標差がそのままモデルのズレになる。／ IfcBuildingStorey＝IFC の「階」を表す要素。

2できること（操作 → 結果）

GHノードではなく、単体で動く Python スクリプト（tkinter GUI）です。起動すると2段階でファイルを選ぶだけ。

操作と結果
操作結果
入【Step1】基準 IFC を選択原点キューブ入りの IFC を基準（ベースモデル）として読み込む。
入【Step2】追加 IFC を複数選択合算したい IFC をまとめて選ぶ。各モデルのオフセットを事前計算する。

3使い方

  1. 各 IFC に原点キューブがあるか確認結合するすべての IFC に、基準位置を示す原点キューブが入っていることを確認する（無いとズレを計算できない）。
  2. ツールを起動するpy C:\Python1\Join_IFC_model\Join_IFC_model.py で起動（IDE 実行でも可）。
  3. 基準 IFC を選ぶ【Step1】のダイアログで原点キューブ入りの基準 IFC を1つ選択。
  4. 追加 IFC を選ぶ【Step2】で合算したい IFC を複数選択。各モデルのオフセットが自動計算される。
  5. ズレを確認して OKGUI に出た座標差（オフセット）を目視で確認し、問題なければ OK。階ごとに補正して結合が走る。
  6. 統合 IFC を後工程へ出力された merged.ifc を ifc2rhino や ライノセラスに変換 へ渡して 3dm 化する。

5実データでの手順

🧱 原点キューブ入りの IFC 2〜3本で試す

専用サンプルはありません。同じ原点キューブを仕込んだ実案件の IFC を数本用意すると、位置合わせの効果が一目でわかります。

  1. 基準＋追加を用意原点キューブ入りの IFC を、基準1本＋追加1〜2本そろえる。わざと原点をずらした状態で用意すると効果がわかりやすい。
  2. 結合して確認起動して Step1／Step2 でファイルを選び、GUI に出た Δ（オフセット）を確認して OK。
  3. 3dm 化して重なりを見る出力の merged.ifc を ifc2rhino で 3dm 化し、各モデルが原点でぴたりと重なっているか Rhino 上で確認する。

6注意点・既知の罠

  • ⚠ 原点キューブが無いとズレを計算できない

    このツールの位置合わせは 各 IFC に入った原点キューブの座標差が頼りです。キューブが入っていない IFC はオフセットを算出できず、正しく合わせられません。結合前に全ファイルへキューブが仕込まれているか確認してください。

  • ⚠ 最速版ゆえ「マテリアル重複チェック」は省略

    v6.1 は速度優先で reuse_identities（マテリアル等の重複チェック）をスキップしています。その分、統合ファイルはやや大きめになり、重複エンティティが残ることがあります。色を正しく持ち込みたい・厳密に統合したい場合は、姉妹版の Join_IF_model（高機能・色転写つき）を使ってください。

  • ⚠ 補正は「階（Storey）単位」。階構成が想定と違うと崩れる

    オフセットは IfcBuildingStorey 単位で適用します。階の切り方がモデル間で食い違っていると、補正が意図とずれることがあります。結合後は必ず 3dm 化などで重なりを目視確認してください。

  • ⚠ ズレ確認ダイアログは飛ばさない

    結合前の GUI ダイアログで Δ（オフセット）を必ず確認してください。値が想定外に大きいときは、原点キューブの位置ミスや単位（mm/m）の食い違いを疑うサインです。

💡 この機能は統合版 IFCMerger にも搭載されています（本ツールは単体で引き続き使用可）。
