図面化パイプライン (Make2D)

通り芯一括抽出 grid_extractor_v2

図面の通り芯（X1・Y3…）を、ブロックの内も外もまとめて自動で拾う。

稼働中 v2（手動追加・除外対応） GH Python ノード

MOTION PREVIEW

動作デモ

PREVIEW⟲ LOOP 10s

図面上の通り芯記号（X1・X2…）をRunで一括抽出して緑でハイライトし、自動で拾い漏れた通り芯だけを手動追加で橙に補う、という流れを10秒でループ再生しています。

1なにをするツール？

2D図面から通り芯（X1・Y3などの基準線）を一気に拾い出すツールです。通り芯の「名称（記号円の中の文字）」「基準点」「符号を囲む円」「一点鎖線」をセットで抽出します。図面のなかで通り芯がブロックの中に入っていても、ブロックの外に生テキストで置かれていても、どちらも拾えるのが強みです。

自動抽出は賢いルール（記号円は「文字を含む最小の閉曲線」、通り芯線は「端点が記号のそばにある最長の直線」）で誤検知を防ぎますが、図面によっては取りこぼしや誤りが出ます。そこで AddIds（手動追加）と DelIds（手動除外）の端子を用意し、最終結果 ＝ (自動抽出 ∪ 手動追加) − 手動除外 という形で人が仕上げられます。ここで拾った通り芯は、下流の 統合JSON生成 に渡って、Make2Dの位置合わせの基準になります。

※ 通り芯＝建物の基準となる縦横のライン（X1, X2… / Y1, Y2…）。図面の位置合わせの「座標の目盛り」に当たります。／ ブロック＝図形をひとかたまりにまとめた部品。中に文字や線が入っていると通常の抽出では拾いにくくなります。

2入出力（GHノードの端子）

入力ポート
端子説明
Run自動抽出を実行する。押した時だけスキャンし、結果はstickyに保持（Run=Falseでも前回結果を出し続ける）。重い再スキャンを避けるため Button 推奨。
AddIds手動で追加したい通り芯のGUID。判定フィルタを通さず強制追加する。自動で拾えなかった通り芯のレスキュー用。touch_picker の出力を接続するのが定番。
DelIds手動で除外したい通り芯のGUID。構成要素（ブロック／文字／円／線）のどれを選んでも、一致するエントリごと除外される。
Reset任意。保持している自動結果とキャッシュを破棄する（Button）。
Highlight任意。未接続＝ノード選択中だけ光る（GH標準プレビュー感覚）／True＝常時表示／False＝非表示。
出力ポート
端子説明
grid_names通り芯の名称（"X1" "Y3" など）。
grid_points各通り芯の基準点（記号円の中心）。
grid_lines通り芯の一点鎖線（円・直線）。DataTreeで明示リスト化され、下流でそのまま使える。

抽出結果はRhinoのビューポートにもハイライト表示されます（自動＝緑・手動追加＝橙。円と直線は太線、基準点は丸マーカー）。

3使い方

  1. 図面を開いてノードを置く通り芯記号が入った2D図面をRhinoで開き、GHにこのノードを配置する。
  2. Runで自動抽出Run に Button を付けて押す。通り芯が緑でハイライトされ、名称・基準点・線が出力される。
  3. 結果を目で確認ビューポートの緑ハイライトを見て、拾い漏れ・誤検知がないかチェックする。
  4. 取りこぼしを手動追加拾えなかった通り芯があれば、touch_picker でGUIDを拾って AddIds へ流す（橙で追加表示）。誤った関連付けも、正しい線を選んでAddIdsに入れると上書きできる。
  5. 余計なものを手動除外通り芯でないものを拾っていたら、2個目のピッカーで選んで DelIds へ。除外が最優先で効く。
  6. 統合JSONへ渡す仕上がった通り芯を GH_SectionDataJSON に接続し、断面符号・レベル線と束ねて統合JSONにする。

4つながり（ワークフロー）

前工程 なし（2D図面から直接抽出する起点ノード）
↓ grid_extractor_v2（このノード） ↓
後工程 統合JSON生成 GH_SectionDataJSON

通り芯・断面符号・レベル線が 統合JSON に集まり、そこから プランナー が切断計画を作ります。全体像は Make2Dパイプライン詳細図 へ。

5サンプルモデルでデモ

📐 サンプルモデル\02_2D図面.3dm

通り芯記号（X1, X2, Y1…）・断面符号（A-A）・タイトル文字入りの2D図面。通り芯抽出・断面符号抽出・統合JSONのデモ用。

  1. 図面を開くサンプルモデル\02_2D図面.3dm を開く。
  2. ノードを置いてRunGHにこのノードを置き、Run のボタンを押す。X1・X2・Y1などが緑でハイライトされる。
  3. 出力を確認grid_names にX1/X2/Y1…が並び、grid_points に各基準点が出ることを確認。
  4. 手動追加を試すわざと1本を除外→AddIdsで戻す、を試すと「自動∪追加−除外」の効き方が体感できる。

6注意点・既知の罠

  • ⚠ Runは必ずButtonで（solve中のモーダル選択はフリーズの元）

    自動スキャンは重いので Run はButtonにして、押した時だけ走らせます。選択UI（ピック）は本体に持たせず、touch_picker を別に置いて AddIds/DelIds に接続する構成が推奨。solve中のモーダル選択はフリーズの原因になります。

  • ⚠ ブロック内の長い横線を通り芯と誤認しやすい

    断面詳細が丸ごとブロック化されたリスト図などでは、ブロック内の長い横線を通り芯線と誤認して交点が全滅することがありました。v2では「線＝端点が記号のそばにある最長の直線」というルールで対策済みですが、変な線を拾ったら DelIds で外し、正しい線を AddIds で差し替えてください（手動が自動を上書きします）。

  • ⚠ 結果はドキュメント単位で保持

    抽出結果は「このコンポーネント × このドキュメント」ごとに保持されます。図面を開き直すと「未計算」に戻るので、もう一度 Run を押してください。
