DWG図面チェック

DWGをまとめて3dm化して、ファイル名とタイトルの食い違いを自動検図——目視チェックの負担をゼロに近づける。

稼働中（手動実行） スタンドアロン2本立て Rhino（ヘッドレス）利用

MOTION PREVIEW

動作デモ

PREVIEW⟲ LOOP 10s

DWGフォルダを一括で3dmに変換しつつ、ファイル名の[ ]表記と図面内タイトルの[ ]表記を突き合わせ、一致=OK・不一致=NGをExcelに一覧出力する流れを10秒でループ再生しています。

1なにをするツール？

DWG図面をまとめて扱う独立した2本のスクリプトをセットにしたものです。どちらもフォルダを指定して手動で走らせる、シンプルな一括処理ツールです。

① DWG一括3dm変換スクリプト——指定フォルダ内の全DWGを順に開き、同じ名前の.3dmとして一括保存します。DWG納品物をRhino作業用の3dmに戻す「下ごしらえ」用です。

② タイトル×記号 検図（batch_check_rhino.py）——DWG図面の中のテキスト（ブロックの中まで再帰的に拾います）を集め、「ファイル名の[ ]表記」と「図面内タイトルの[ ]表記」が一致しているか、さらに指定した座標エリア（右上・左下）のテキスト同士が一致しているかを OK / NG で判定し、Excel（またはCSV）に一覧出力します。「ファイル名と図面の見出しがちゃんと合っているか」を目でひとつずつ確認する手間を自動化する検図ツールです。

※ 検図＝図面の記載が正しいか、食い違いがないかを点検する作業。／ [ ]表記＝図番や部材符号を角カッコで囲んで示す社内ルール（例: S31C-X11Y3a）。

3使い方

  1. 対象フォルダを用意する検図したいDWG図面を1つのフォルダにまとめておく。
  2. （必要なら）先に3dm化するRhino作業用に3dmが要る場合は 01_DWG一括3dm変換スクリプト.py を走らせて一括変換。検図だけなら省略可。
  3. 照合エリアを設定する座標照合を使う案件では、SYMBOL_AREA（右上）と MARK_AREA（左下）の座標をコード内で案件に合わせて指定する。
  4. 検図を走らせるbatch_check_rhino.py を実行。各DWGを開いてテキストを集め、[ ]表記と座標エリアを突き合わせる。
  5. 結果Excelを確認する出力された _check_result.xlsx を開き、NGの行だけを拾って図面を直す。

4つながり（ワークフロー）

この2本は単独で使えます。他ノードへの自動連携はなく、DWGフォルダを渡すだけで完結する手動実行ツールです。目的の近い 断面符号＆タイトル抽出 や 方位文字一括修正 とは別実装で、直接のデータ受け渡しはありません。

5実際に試すには

🗂️ 検図したいDWGフォルダをそのまま使う

DWG図面を扱うツールのため専用サンプルモデルはありません。[ ]表記のファイル名が付いたDWGフォルダで試すのが手軽です。

  1. テスト用フォルダを複製対象DWGをコピーしたフォルダで試す（原本は残す）。
  2. まず[ ]照合だけで走らせる座標照合を使わないなら、ファイル名と図面タイトルの[ ]一致だけでも十分に効果あり。
  3. 結果Excelを開くNG行が正しく拾えているか、実ファイルと突き合わせて確認する。

6注意点・既知の罠

  • ⚠ 照合座標が既定で全て0＝そのままだとデバッグモード

    検図スクリプトの座標エリア（SYMBOL_AREA / MARK_AREA）は初期値が全て0のため、そのままだとデバッグモードで動きます。座標照合を本番で使う場合は、案件ごとにコード内の数値を手で編集する運用が前提です。

  • ⚠ テキスト収集にRhinoを使う

    ブロック内まで含めた図面テキストの収集にRhino（ヘッドレス）を使います。Rhinoが入った環境で実行してください。純粋な事務PCでは動きません。

  • ⚠ 手動実行のスタンドアロン2本立て

    2本のスクリプトはそれぞれ独立していて、自動連携はありません。必要な方を手で走らせてください（3dmが要らなければ変換スクリプトは不要）。

  • ⚠ フォルダに紛れ込んだJSONは無関係

    作業フォルダにMake2D統合JSON形式のファイルが紛れ込んでいることがありますが、この検図スクリプトとは直接の連携はありません（迷い込みサンプルの可能性）。判定結果に影響はしません。
