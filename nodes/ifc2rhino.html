IFC⇔Rhino変換 ifc2rhino

Teklaの孔を壊さずRhinoへ丸ごと持ち込み、属性ごとIFCへ書き戻す——IFC変換の本命。

稼働中 チーム配布済 Rhino8 / ifcopenshell IFCIMP・IFCINFO・IFCEXPORT

MOTION PREVIEW

動作デモ

PREVIEW⟲ LOOP 10s

孔あきのIFCを孔を壊さず変換 → 属性を焼き込んでRhinoへ → 編集後はIFCへ往復書き戻し、という流れを10秒でループ再生しています。

1なにをするツール？

Tekla等が書き出したIFCファイルを、ボルト孔や切欠きを壊さずにRhino8へ取り込む「IFC→3dm変換の本命」です。ifcopenshell（OpenCASCADEカーネル）でIFCのブーリアン演算を直接解くので、Rhino標準の取込プラグインで起きがちな「孔がふさがる／形が破綻する」が起きません。

取り込んだ部材はメッシュ＋ブロック（Teklaアセンブリ）＋IFC属性（GlobalId・Pset・数量など）付きでベイクされます。属性は IFCINFO で部材をクリックするだけで一覧表示。編集後は IFCEXPORT で.3dmを再びIFC（IFC2X3/IFC4）へ書き戻せて、IFC→3dm→IFCの往復でも情報が保持されます。変換のたびにQCレポート（HTML）が自動生成され、孔破綻や重量照合を確認できます。

※ IFCIMP＝取込／IFCINFO＝属性ビューア／IFCEXPORT＝書き戻し の3コマンド一式（Rhino8専用・Rhino7では動きません）。焼き込んだ属性はそのまま ifc_attr_search_v2 の検索対象になり、Make2Dパイプラインへつながります。

3使い方

  1. 初回セットアップsetup_aliases.py をRhino8にドラッグ&ドロップ実行。IFCIMP/IFCINFO/IFCEXPORT のエイリアスとツールバーが登録されます（冪等・配置場所を自動検出）。
  2. 取り込むRhinoで IFCIMP → .ifc選択 → 現在ドキュメントへベイク。QCレポートが自動で開きます。大量なら .ifc を IFC変換_ドロップ.bat にドラッグして無人変換。
  3. 中身を確認するIFCINFO で部材クリック → GlobalId/Pset/数量などを一覧。QCレポートで孔破綻や重量照合（誤差5%閾値）をチェック。
  4. パイプラインへ焼き込まれた属性は ifc_attr_search_v2 の検索対象。本体／附帯を仕分けてMake2Dへ渡せます。
  5. IFCへ書き戻す編集後 IFCEXPORT → スキーマ（IFC4推奨）・出力レイヤーを選んで .ifc 出力。往復で属性が保持されます。

4つながり（ワークフロー）

ifc2rhino（このツール） ↓
後工程 IFC属性検索 ifc_attr_search_v2

前段にIFCの結合・スキーマ変換（Join_IFC_model / IFC_Converter 等）を置くこともできます。取り込んだ属性はMake2Dパイプライン全体で活用されます。

5実データでの使いどころ

🧱 Tekla IFC → Rhinoで検図・製作図化

TeklaのIFC（Bolts:Off想定）をRhinoへ取り込み、属性ごと検図してMake2Dで製作図に落とす、という実運用の流れです。

  1. IFCを書き出すTeklaからIFC（IFC2X3/IFC4）を書き出します。
  2. .3dm化して確認IFCIMP または ドロップ.bat で.3dm化。QCレポートで孔・重量照合を確認。
  3. 仕分けて製作図へIFCINFOで属性確認 → ifc_attr_search_v2 で本体/附帯を仕分け → Make2Dで製作図化。
  4. IFCへ往復修正が出たら IFCEXPORT でIFCへ書き戻し、上流と往復します。

6注意点・既知の罠

  • ⚠ CLI版はブロック化できない（rhino3dmのバグ）

    CLI版は rhino3dm の InstanceDefinitionTable.Add にヒープ破壊バグがあるため、ブロック化なしのフラット出力専用です。ブロック（アセンブリ）が要るときは必ずRhino8内の IFCIMP を使ってください。

  • ⚠ Rhino8専用（Rhino7非対応）

    Rhino7では動きません。フォルダ内の IFC.7.rui は出所不明の副生成物の疑いで、どのスクリプトからも参照されていません（削除前に一応確認を）。

  • ⚠ 属性抽出ロジックが3ファイルに重複

    ifc_import_rhino.py / ifc2rhino.py / ifc_export_core.py にほぼ同じ属性抽出ロジックが重複実装されています。仕様を直すときは3ファイル横断で確認してください。

  • ⚠ venv312 は依存一式で4000ファイル超

    同梱の venv312 はifcopenshell等の依存で非常に多数のファイルを含みます。フォルダ移動やバックアップは重くなるので、扱うときは直下の.pyだけを対象にするのがコツです。
