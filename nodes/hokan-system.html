柱詳細図補完 hokan_system

手薄な柱詳細図の抜け漏れをAIが数えて、位置寸法と名称注記を自動で埋める。

稼働中（実機検証済） シートビルダー v3.1 / ベイク v5 3段パイプライン

動作デモ

素っ気ない柱詳細図 → 不足をスキャンして数える → 位置寸法と名称ラベルを自動で書き足す、という3段を10秒でループ再生しています。

1なにをするツール？

寸法も名称注記もほとんど入っていない手薄な柱詳細図を、完成品レベルの図面へ底上げする補完システムです。もともとはある工区の柱詳細図51面を、完成形サンプル（隣接工区・協力会社の完成図面）と同じくらいの表現力に仕上げるために作られました。実データでは「周辺・附帯部材の取付位置寸法や名称注記が全図面で0件」といった抜けを埋めるのが狙いです。

処理は3段のパイプラインです。まず ①ギャップスキャンで「何が足りないか（位置寸法・名称が無い箇所）」を数え上げてレポート化。次に ②補完注記バッチで、不足していた取付位置の寸法と部材名称のラベルを自動で書き込みます（寸法は「補完_寸法」、名称は「補完_注記」という専用レイヤーに分けて入るので、後から見分け・手直しがしやすい）。最後に ③補完前後の比較PDFを作り、どこが追加されたかを目で確認できます。

※ ギャップスキャンはRhinoを起動せずPython（py -3.12）だけで走るので軽快。補完注記はRhino上のRunPythonScriptで実行し、GHノード GH_HokanAnnotate からもボタンひとつで動かせます。

3使い方

  1. 対象フォルダを用意する補完したい柱詳細図の.3dmをフォルダにまとめる。いきなり本番ではなく複製で試すのが安全（同梱の test_data サンドボックスが参考になる）。
  2. ギャップスキャンで数えるhokan_gap_scan.py を py -3.12 で実行。Rhino不要。hokan_gap_report.csv/.json に「どこに何が足りないか」が出る。
  3. レポートを確認する位置寸法・名称の不足件数をチェック。おかしな検出が無いか眺めてから次へ。
  4. 補完を実行するrun_hokan_batch.ps1 -Folder … でまとめて、または GH_HokanAnnotate の Run で1枚ずつ。補完_寸法（マゼンタ）と補完_注記（オレンジ）のレイヤーに書き込まれる。
  5. 前後比較PDFで検図する補完前後比較.pdf を開き、追加された寸法・名称が妥当か確認。★要確認ラベルが付いた箇所は人の目で最終判断。

5実データでためす

📁 専用サンプルはなし — 実データ（複製）で試します

本システムは特定案件（南③工区の柱詳細図）向けに作られており、同梱の test_data フォルダに南③工区を複製した .3dmbak サンドボックスがあります。まずはそこで動きを確かめるのが安全です。

  1. サンドボックスで練習C:\Python1\図面整理\hokan_system\test_data\ の複製データを使う（本番の柱詳細図は触らない）。
  2. スキャン→補完→PDFギャップスキャンで不足を数え、補完注記で書き込み、前後比較PDFで結果を確認する3段をひと通り回す。
  3. 検証実績を確認南③工区51面のうち50面で検証済み（Y1C-17aの1面のみ未処理とREADMEに明記）。まずはこの実績範囲で挙動を体感する。

6注意点・既知の罠

  • ⚠ 特定案件向け・本番は複製で試してから

    もともと南③工区の柱詳細図に合わせて作られたシステムです。別案件にそのまま当てると検出ルールが合わないことがあります。本番ファイルをいきなり書き換えず、必ず複製（test_data 相当）で挙動を確認してから使ってください。

  • ⚠ ★要確認ラベルは人の目で最終判断

    自動補完はあくまで下書きです。確信の低い箇所には★要確認ラベルが付きます。前後比較PDFで必ず検図し、正しくない補完は手直ししてください。

  • ⚠ RunPythonScriptの文字コード宣言に注意

    補完注記はRhinoの RunPythonScript から呼びますが、この経路はスクリプト先頭の coding 宣言を無視するという既知の癖があります。日本語を扱うスクリプトはこの前提で書かれている必要があり、自作で差し替える際は文字化けに注意してください。

  • ⚠ 位置合わせは1:1作図＋統合JSONアンカーが前提

    補完注記が正しい場所に入るのは、図面が1:1で作図され、統合JSONのアンカー（基準）と整合しているからです。縮尺やアンカーがずれていると寸法・ラベルの位置がずれます。上流のMake2D出力が正しく配置されていることが前提です。
