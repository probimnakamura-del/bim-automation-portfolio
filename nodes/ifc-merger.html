1なにをするツール？

社内に散らばっていた 3系統の IFC 結合ツール（Join_IFC_model / Join_IF_model / IFC編集ツール）の機能を、ひとつの Web UI（ブラウザ画面）にまとめた統合版です。ブラウザで start.bat を起動すると http://localhost:8620 が開き、結合・範囲抽出・レポート履歴の3タブを行き来できます。CLI（ifc_merger.py）でも同じ処理を回せます。外部通信なしの完全ローカル動作です。

① 結合：基準1ファイル＋追加を複数まとめて、基準の座標系にそろえて1つに統合します。各追加ファイルの原点キューブを自動検出（スコアリング）して基準キューブとの差からオフセットを自動算出し、Tekla を自動判定して色・レイヤーを非Tekla側へ転写。同じ GUID が衝突したときは体積と重心を比較して形の食い違いを検出し、方針（基準を残す／新規を残す／両方を GUID 再発行で残す）を選べます。

② 範囲抽出：1つの IFC から 直方体（bbox）と交差する要素だけを、Pset・材料・Assembly・空間構造を保ったまま新しい IFC に取り出します。数値 bbox 入力が主で、フットプリント（平面配置の SVG）から矩形ドラッグでも指定できます。③ レポート：結合・抽出ごとに、ファイル別統計・オフセット・転写件数・GUID 衝突表・整合性検査をまとめた HTML レポートを残します。

※ IFC＝建物の3Dモデルを部材属性ごとやり取りする業界標準ファイル。／ GUID＝各要素に振られる一意のID。同じGUIDが両ファイルにあると「同一要素」とみなして形状を照合できる。／ 原点キューブ＝各モデルの基準位置を示す小さな立方体で、その座標差がモデルのズレになる。

3使い方

  1. Web UI を起動するIFCMerger\start.bat をダブルクリック。ブラウザで http://localhost:8620 が開く（完全ローカル）。CLI派は venv\Scripts\python.exe ifc_merger.py base.ifc add1.ifc -o out.ifc。
  2. 結合：IFCをドラッグ＆ドロップ結合タブに複数のIFCを投入。ファイルごとにスキーマ・単位・要素数・Tekla判定・原点キューブ検出結果が表示される。
  3. 基準を選び、必要なら上書き基準ファイルを1つ選択。ファイル別に「原点キューブ自動」ON/OFF・手動オフセット・Z回転を上書きできる。GUID衝突の方針を選んで「結合を実行」。
  4. 結果とレポートを確認完了後、統合IFCのダウンロードとHTMLレポート表示。形状不一致があればレポートの衝突表で確認する。
  5. 範囲抽出：bboxで切り出す範囲抽出タブで対象を選び、数値 bbox(mm) を入力（フットプリントSVGで矩形ドラッグ指定も可）。「抽出を実行」で交差要素だけの新IFCを書き出す。
  6. 後工程へ渡す出力IFCを ifc2rhino や ライノセラスに変換 へ渡して .3dm 化する。

4つながり（ワークフロー）

統合元 Join_IFC_model（原点キューブ結合） Join_IF_model（Tekla色転写結合） IFC編集ツール（範囲抽出） IFC_Converter（スキーマ変換）
↓ IFCMerger（このツール） ↓
後工程 IFC⇔Rhino変換 ifc2rhino IFC自動変換パイプライン（ライノセラスに変換）

これらの機能を統合したのが本ツールです。移植元の Join_IFC_model / Join_IF_model / IFC編集ツールは無改変で温存されており、単体版も引き続き使えます。スキーマ（IFC2X3/IFC4）がそろっていないファイルは、先に IFC_Converter でIFC4化してから結合してください。どれを使うか迷ったら IFCツール使い分けガイド をどうぞ。

5実データでためす

🧩 原点キューブ入りの IFC 2〜3本（Tekla ＋ 意匠など）

専用サンプルはありません。同じ原点キューブを仕込んだ実案件のIFCを数本用意すると、位置合わせと色転写の効果が一目でわかります。テスト22件＋回帰スイートは緑で通っています。

  1. 基準＋追加を用意原点キューブ入りのIFCを、基準1本＋追加1〜2本。Tekla側（色つき）を混ぜると転写の効果が見える。
  2. 結合してレポートを見るstart.bat で起動し結合を実行。HTMLレポートでオフセット・転写件数・GUID衝突を確認する。
  3. 範囲抽出も試す統合IFCに対し bbox を指定して1フロア分だけ抽出し、Pset や材料が残っているかビューアで確認する。

6注意点・既知の罠

  • ⚠ mm前提・スキーマ一致必須（不一致はハードエラー）

    オフセット・許容差はすべて mm 前提です。LengthUnit が食い違う／schema（IFC2X3・IFC4等）がそろっていないファイル同士は結合を中断します（換算機能はありません）。そろえるには先に IFC_Converter でIFC4化してください。

  • ⚠ モードC（非Tekla同士）はGUID再発行で両方保持

    非Tekla同士の結合ではGUIDの信頼性が低い前提のため、体積比較を行わず、衝突GUIDは再発行して両方保持します（色マップも無し・警告表示）。同一要素の重複が残ることがあります。

  • ⚠ 壁開口（IfcRelVoidsElement）は再構築しない

    結合コピーでは IfcOpeningElement を対象外とし、開口の関係も再構築しません（移植元のJoin系と同じ挙動）。Tekla構造モデルは部材形状自体に孔が焼き込まれているため影響しませんが、意匠系IFCの壁開口は結合後に失われる可能性があります。

  • ⚠ フットプリントSVGは「部分表示」になり得る

    範囲抽出のフットプリントは要素数上限（3000）とタイムアウト付きです。上限・タイムアウト時は計算済み分だけを描画して「部分表示」を警告します。厳密な範囲指定は数値bbox入力で行ってください。

  • ⚠ 起動は同梱venvのpythonで（storeスタブに注意）

    CLIは python ifc_merger.py ではなく venv\Scripts\python.exe を直接指定します（Windowsのストア版Pythonスタブに当たって失敗するため）。ポートは 8620 固定。venv は再作成しないでください（Python 3.14 用のビルド済み ifcopenshell が入っており再取得が困難）。
