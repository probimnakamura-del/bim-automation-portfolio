かさ上げ鋼材 7分割クラスター版 kasage_nodes

同じかさ上げ鋼材づくりを、7つのノードに分けてパイプライン化した版。途中に別処理を挟める。

稼働中（姉妹実装） 7分割クラスター版 GH Python3 ノード

MOTION PREVIEW

動作デモ

PREVIEW⟲ LOOP 10s

既存梁＋通り芯を先頭ノードに入れると、データが [1]→[7] を数珠つなぎで流れ、各工程が順に点灯して、最後にかさ上げ鋼材がベイクされる——という流れを10秒でループ再生しています。

1なにをするツール？

単一ファイル版の kasage_steel とまったく同じ「梁の上にかさ上げ鋼材を自動で乗せる」処理を、1本の巨大スクリプトではなく7個のGHPythonコンポーネントに分割し、クラスター化して使う姉妹実装です。

FL基準取得 → 断面選定 → 2Dプロファイル → レール補正 → ソリッド化 →（Split／Boolean後の）メタ再スタンプ → ベイク、という7段パイプライン。各ノードは sc.sticky や ghenv に依存しない純粋関数寄りの設計なので、ソリッド化と再スタンプの間に部材カットなどの別処理を割り込ませても、UserStringに刻んだ材質・断面が子形状へ引き継がれます。断面カタログとFL取得ロジックは単一版と共通です。

※ クラスター＝複数のGHコンポーネントを1つにまとめて部品化したもの。UserString/UDA＝オブジェクトに紐づく属性（材質・断面名など）。「1ノードで済ませたい」なら単一版、「工程の途中に割り込みたい」なら本7分割版が向きます。

2入出力（7工程のパイプライン）

[1]〜[5] がクラスター内、[6][7] は分割（Split／Boolean）処理の後段です。各ノードは前ノードの出力を入力名で受け取ります。

7段パイプラインの流れ
工程役割主な入力 → 出力
[1] beam_data通り芯からFL取得＋梁レール抽出Base_Ref / Beam_Obj / Is_Main → FL_Z / Top_Z / Rails / Center / Rail_Dir
[2] section_select主／次梁ルールで断面自動選定＋かさ上げ量計算[1]のFL・Top_Z / Material / Section_Override / Special_Name → Section_Params / Material / Section_Name
[3] profile断面パラメータから2Dプロファイル生成[2] Section_Params → Profile
[4] rail_fixセットバック等でレールを補正[1] Rails / Rail_Dir・Dist_S/E・Z_Offset_S/E → Adjusted_Rails
[5] solidプロファイル×レールでソリッド化＋UDA刻印[3][4]統合 → Steel_Breps（Material／Section内包）
[6] restampSplit／Boolean後の子形状へUDAを再継承[5] Parents ＋ 分割後 Children → Out_Geo / Out_Material / Out_Section
[7] bakeドキュメントへベイク（属性UDA付き）[6]出力 → Baked_Ids / Log

3使い方

  1. 7ノードを取り込むkasage_nodes\1〜7 を NodeGen 等でノード化し、[1]→[7] の順に配線（クラスター化推奨）。
  2. FL基準と梁を渡す[1] の Base_Ref に通り芯ブロック、Beam_Obj に梁、Is_Main で主／次を指定。
  3. 断面・材質を決める[2] で Material / Section_Override / Special_Name を指定（未指定は自動選定）。
  4. 逃げを調整[4] で Dist_S/E と Z_Offset_S/E を設定。
  5. ソリッド生成[5] で Steel_Breps が出る。ここまでがクラスター内。
  6. （必要なら）分割処理を挟む[5] と [6] の間に 部材カット やブーリアンなどの別処理を割り込ませられる。
  7. 再スタンプ → ベイク[6] で子形状へ材質／断面を継ぎ直し、[7] で Bake。属性UDA付きでドキュメントへ焼き込む。

4つながり（ワークフロー）

単独で使えます。 [1]〜[7] のクラスター内で「梁上かさ上げ鋼材の生成→ベイク」が完結します。

成果物は単一ファイル版の kasage_steel と同じです。1ノードで手早く済ませたいなら単一版、ソリッド化とベイクの間に分割・カットを挟みたいなら本7分割版を選びます。取り込みには NodeGen、途中のカットには 部材カット自動延長ツール が便利です。

5試し方（実データ）

🧩 梁＋通り芯ブロックのあるモデルで

専用サンプルはありませんが、単一版と同じ入力（梁と通り芯ブロック）で動くため、サンプルモデル\01_柱梁フレーム.3dm をそのまま流用できます。

  1. 7ノードを並べる[1]〜[7] をノード化して数珠つなぎに配線（クラスター化しておくと再利用しやすい）。
  2. [1] に入力通り芯ブロックを Base_Ref、梁を Beam_Obj に接続。Log でFL取得元を確認。
  3. [5] まで流すソリッド（Steel_Breps）が出るところまで確認。分割処理を試すなら [5]/[6] 間に挟む。
  4. [7] でベイク最初は材質・断面が正しくUDAに入っているか [6]/[7] の Log で確認してからベイク。

6注意点・既知の罠

  • ⚠ 再スタンプ [6] が肝。分割は必ず [5] と [6] の間へ

    Split／Boolean を通すと子形状は親のUserString（材質・断面）を失います。[6] で親から子へUDAを継ぎ直さないと属性が消えます。分割・カット処理は必ず [5] のあと、[6] の前に挟んでください。

  • ⚠ 断面カタログは単一版と共有

    断面辞書・FL取得ロジックは kasage_steel と共通です。どちらが最新の正本かはコードに明記がないため、両者の断面定義がずれていないかは念のため確認してください。

  • ⚠ ブロックはGHの端子を「型」として渡せない

    [1] の Base_Ref にはブロックをそのまま型で運べません。GUID／ジオメトリで渡し、受け側が展開します。参照モデルの通り芯を拾えないときはここを疑ってください。

  • ⚠ GH Python3 ノードとして使う

    7ノードはRhino 8のPython3コンポーネント前提です。クラシックなIronPython 2.7コンポーネントでは動かない場合があります。NodeGen で取り込むとPython3ノードとして生成されます。
