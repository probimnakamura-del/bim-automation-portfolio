統合JSON生成 GH_SectionDataJSON

図面の全抽出結果を1本のJSONに束ねる——パイプラインの結節点。

稼働中 GH Python ノード パイプラインの中核

MOTION PREVIEW

動作デモ

PREVIEW⟲ LOOP 10s

通り芯・断面符号・レベル線の3系統が中央のハブへ吸い込まれ、交点を判定して1本の統合JSONに束ねられる流れを10秒でループ再生しています。

1なにをするツール？

図面から別々に拾ってきた4つの情報——表題欄・断面符号・通り芯・レベル線——を1本のJSONにまとめる中核ノードです。上流の抽出ノードたちがそれぞれ集めたバラバラの結果を、ここが共通の言葉（統合JSON）に翻訳して束ねます。以降の工程は全部この1ファイルを見れば済むようになる、パイプラインの結節点です。

ただ集めるだけでなく、「どの通り芯とどの通り芯が交わるか」を計算して交点に名前（X1-Y1 など）を付け、断面がどちらを向いているか（Make2Dの向き）まで解析します。交点判定はすべて2Dに投影してから行い、線が少し足りずに交わらない所は ExtTol 端子で「線を仮想的に伸ばして交差を拾う量」を調整できます。出力の Out_JSON には Sections（断面）・Intersections（交点）・ProductKey（製品記号）などがまとまります。

※ 統合JSON＝この先の全ノードが共通で読む「図面の設計データ台帳」。／ ProductKey＝製品記号。あとでIFC属性検索がモデルを探すときの手がかりになります。

3使い方

  1. 通り芯をつなぐgrid_extractor_v2 の出力を GridNames / GridLines / GridPoints へ。
  2. 断面符号・タイトルをつなぐGH_SectionSymbolTitle の出力を In_Names / In_Lines_H / In_Points / In_TitleNames へ。
  3. レベル線をつなぐGH_LevelLines の出力を LevelNames / LevelLines へ。1FL・2FLの高さ基準がそろう。
  4. 交点が出るか確認solveして Out_JSON の Intersections を見る。交点がゼロなら次のステップ。
  5. ExtTolで延長許容を調整通り芯線がわずかに届かず交わらないときは ExtTol を少し上げて、線を伸ばして交差を拾わせる。
  6. 下流へ渡すできた統合JSONを section_lower（小文字断面）と GH_Make2DPlanner へ。

4つながり（ワークフロー）

前工程 通り芯一括抽出 grid_extractor_v2 断面符号＆タイトル GH_SectionSymbolTitle
↓ 統合JSON生成 GH_SectionDataJSON（このノード） ↓
後工程 小文字断面 section_lower Make2Dプランナー GH_Make2DPlanner

レベル線（GH_LevelLines）も入力の1つです。全体像は Make2Dパイプライン詳細図 で確認できます。

5サンプルモデルでデモ

🧩 サンプルモデル\02_2D図面.3dm

通り芯記号（X1・X2・Y1…）・断面符号（A-A）・タイトル文字入りの2D図面。通り芯抽出・断面符号抽出・統合JSONのデモ用。

  1. 図面を開くサンプルモデル\02_2D図面.3dm を開く。
  2. 上流を回す通り芯抽出と断面符号抽出を先にsolveし、それぞれの出力を本ノードへ配線する。
  3. 統合してJSONを見る本ノードをsolveし、Out_JSON に Sections と Intersections（X1-Y1 など）がそろうのを確認。
  4. 交点ゼロなら ExtTol交点が空なら ExtTol を少し上げて再solve。線の届かなさが原因かを切り分ける。

6注意点・既知の罠

  • ⚠ 交点ゼロの主因は「3D距離での判定」— 本ノードは全2D投影で解決済み

    過去、通り芯どうしの交点が1つも出ない不具合の原因は、高さ（Z）の違う線を3Dの距離で交差判定していたことでした。本ノードはすべて2Dに投影してから交点を取る設計に直してあります。それでも交点が出ないときは、まず線が本当に足りているかを疑ってください。

  • ⚠ 線がわずかに届かないときは ExtTol を上げる

    通り芯線の端がほんの少し交点に届かず交差しないことがあります。その場合は ExtTol（延長許容）を少し上げて、線を仮想的に伸ばして拾わせてください。上げすぎると意図しない交点を拾うので、少しずつ調整します。

  • ⚠ ここが結節点 — 上流4系統がそろって初めて完成する

    通り芯・断面符号・レベル線・表題欄のどれかが欠けると、統合JSONもその分だけ欠けます。下流（section_lower・プランナー）が期待通り動かないときは、まず本ノードの入力4系統がすべて来ているかを確認してください。
