BIM-Shield IFC⇔Rhino 品質保証パイプライン

TeklaのIFCとRhinoモデルを行き来させ、部材を自動仕分けして検証レポートまで出す、変換の関所。

稼働中（現行v2.7 / v3.2） run_bim_shield_v27.bat 起動 Rhino.Compute 必須

動作デモ

PREVIEW⟲ LOOP 10s

Teklaが出力したIFCを取り込み、部材を3段階のレイヤー階層にツリー状へ自動分類して、Rhinoの.3dmと検証Excelレポートを吐き出す流れを10秒でループ再生しています。

1なにをするツール？

Tekla（鉄骨のBIMソフト）が出力したIFCと、Rhinoの.3dmを双方向に変換し、その途中で部材を自動仕分け・幾何検証・レポート化までやってくれる「変換の関所」です。IFCを取り込むときは、部材を IfcType（種別）→ ProductName（呼び名）→ Profile_Material_Class（断面・材質） の3段階レイヤー階層に自動分類し、Rhinoで扱いやすい形に整えます。

逆に、Rhinoで作った.3dm（レイヤーやUserTextで属性を持たせたモデル）からIFCを書き出す方向にも対応。Tekla由来の属性（Pset_Tekla_Attributes など）を補完し、Solibri（IFCの検証・納品ツール）向けにデータを整えます。変換結果はExcel/PDFの検証レポートとして出るので、「モデルがちゃんと変換・分類できたか」を目で確認できます。

※ 現行は「IFC→3dm」が v2.7、「3dm→IFC」が v3.2。run_bim_shield_v27.bat が最新エントリを呼びます。旧世代（v2.0/2.1/2.5、旧Eto UIパネル）はコードとして残っていますが現行からは呼ばれず、事実上凍結です（v2.0のエンジンだけは3dm→IFC変換で継続利用）。

3使い方

  1. Rhino.Compute を用意3dm→IFC方向は幾何解析に Rhino.Compute（compute.frontend.exe / localhost:5000）が必須。BIM-Shieldが自動起動しますが、初回は立ち上がりを少し待つ。
  2. IFCを取り込むTeklaが出力した .ifc を run_bim_shield_v27.bat へドラッグ&ドロップ。コンソールに3層レイヤー階層がツリー状に構築されていく。
  3. 分類結果を確認部材が IfcType → ProductName → Profile_Material_Class の階層に並ぶ。変換済み.3dmとして開いてレイヤー構成をチェック。
  4. 検証レポートを見る出力された 検証レポート.xlsx で、幾何検証や属性補完の結果を確認。取りこぼしがないか目視する。
  5. 逆方向（3dm→IFC）Rhinoで属性（UserText）を付けたモデルからIFCを書き出す場合は、v3.2のエクスポータ経路で実行。Solibri納品前のデータ整備に使う。

4つながり（ワークフロー）

単独で使える変換ツールです。 外部の Tekla（IFC出力元）から受け取り、Solibri（検証・納品先）へ渡す関所として働きます。Rhino.Compute を裏で使います。なお、目的がよく似た ifc2rhino が別フォルダに存在し（統合・重複の可能性あり）、監視ダッシュボードの BIM_Dashboard とは現状コード連携がありません。

🛡️ 手元のTekla出力IFCで試す

専用サンプルは同梱していません。実案件のTekla出力IFCで動作を確認します。

  1. 小さめのIFCを選ぶまずは部材数の少ないIFCで試すと、3層レイヤーの構築が追いやすい。
  2. batへドロップrun_bim_shield_v27.bat にドラッグして実行。Rhino.Computeの起動を待つ。
  3. 結果を突き合わせる出力.3dmのレイヤー階層と、検証レポート.xlsxの分類・検証結果を照合。

6注意点・既知の罠

  • ⚠ Rhino.Compute が動いていないと変換できない

    3dm→IFC方向の幾何解析は外部プロセス Rhino.Compute（localhost:5000）に依存します。自動起動しますが、立ち上がりに時間がかかったり、ポートが塞がっていると失敗します。うまくいかないときはまずComputeの状態を確認してください。

  • ⚠ 現行エントリは v2.7 / v3.2。旧版は凍結

    起動は必ず run_bim_shield_v27.bat から。v2.0/2.1/2.5 や旧Eto UIパネル（plugin_main.py）はコードが残っていますが現行からは未importで凍結です（例外的に v2.0 エンジンだけ 3dm→IFC で継続利用）。古いスクリプトを直接叩かないでください。

  • ⚠ ifc2rhino と目的が酷似（統合・重複の可能性）

    別フォルダの ifc2rhino が「Tekla IFC → Rhino変換」というほぼ同じ目的を持っています。どちらを正とするかは未整理のため、混在させる前に用途を確認してください。

  • ⚠ 空フォルダ／使い捨てスクリプトが同居

    フォルダ内に file / runs / 3dm / IFC / Rhino などの空フォルダ（ダイアログ処理の誤生成痕）や、_diag_cone.py・_syntax_check.py といった使い捨て診断スクリプトが残っています。現行の動作には無関係なので触らないでください。
